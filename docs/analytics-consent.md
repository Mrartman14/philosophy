# Аналитика и согласие (consent)

> Статус: **аналитика отключена** (2026-06-20). Решение, а не баг.

## Что и почему

Yandex Metrika ([src/components/yandex-metrika/yandex-metrika.tsx](../src/components/yandex-metrika/yandex-metrika.tsx))
**отключена** и грузится только при явном включении через env-флаг
`NEXT_PUBLIC_ANALYTICS_ENABLED=1`. Код не удалён — отключён.

Причина: трекинг без согласия пользователя — юридический риск (GDPR + 152-ФЗ),
особенно `webvisor` (запись сессий). Пока в приложении нет механизма согласия,
аналитика остаётся выключенной. `webvisor` убран из конфигурации совсем.

**Scope заметки — только сторонняя аналитика (Yandex).** Web-push
(`pushManager.subscribe`, тумблер в настройках) — отдельная per-feature история:
пользователь включает его сам, это не покрывается этим решением.

## Как включить обратно — безопасно (blueprint)

Просто выставить `NEXT_PUBLIC_ANALYTICS_ENABLED=1` **нельзя без consent-механизма**.
Перед возвратом аналитики нужно реализовать opt-in согласие. Принятые решения:

- **Модель — opt-in.** Ничего аналитического не грузится до явного «да».
- **`webvisor` — off** (или за отдельным явным тумблером, не по умолчанию).
- **Consent гейтит только аналитику.** Функциональные/необходимые куки
  (`token`, `appearance`, `locale`) — это «strictly necessary», согласия не требуют,
  ставятся сразу, SSR-read, без FOUC. Consent их не трогает.
- **Cookie-first.** Согласие хранится в куке `consent` (зеркало паттерна
  appearance/locale: SSR-read → provider → запись по действию). Работает анонимно.
- **Бэкенд-хранение согласия — опционально**, не для UX (куки 1 год уже не
  переспрашивают на том же устройстве), а для кросс-девайса залогиненных и
  proof-of-consent/аудита. Бэк не заменяет куку, только дополняет. Поля под
  согласие на бэке сейчас нет — это отложенный gated-seam.

Реализация консента (баннер + provider + `/privacy` + тумблер в настройках)
не делается сейчас, т.к. гейтить нечего — аналитики нет. Эта заметка — её чертёж.

## Связанное

- CSP: пока аналитика off, `script-src` почти весь `'self'` (домен `mc.yandex.ru`
  добавится в политику только при возврате Метрики).
