# Аннотации документов на публичной странице лекции

**Дата:** 2026-06-25
**Статус:** одобрено (дизайн)
**Область:** `src/app/lectures/[id]/page.tsx`, `src/features/lectures` (новый inline-рендер + селектор), переиспользование `src/features/annotations` (`DocumentAnnotations`) и `@/components/annotation-layer` без изменений.

## Проблема

На странице документа `/documents/[id]` пользователь может оставлять аннотации (Word-style маргиналии). На публичной странице лекции `/lectures/[id]` документы сейчас **не рендерятся** — только список ссылок ([LectureDocumentsSection](../../../src/features/lectures/ui/lecture-documents-section.tsx)), а её лейаут — центрированный `max-w-3xl`, мимо грид-полей. Нужно: при просмотре конкретного документа лекции **на самой странице лекции** уметь так же аннотировать.

## Одобренные решения

- **Страница:** публичная `/lectures/[id]` (для читателей), не admin-карточка.
- **Охват:** все документы лекции (любой можно открыть и аннотировать).
- **Привязка аннотаций:** к ДОКУМЕНТУ (`getAnnotationsFor("document", id)`) — те же аннотации, что на `/documents/[id]` (общие, без изменений бэка).
- **Лейаут:** через **маргиналии** (правое поле грида), страница приводится к **общему `.page-grid`-лейауту** (как `/documents/[id]`).
- **Ленивость сохраняется** через **URL-driven выбор документа** (`?doc=<id>`): сервер на запрос рендерит только активный документ + его аннотации.

## Архитектура

### 1. Лейаут страницы → общий `.page-grid`

[/lectures/[id]/page.tsx](../../../src/app/lectures/[id]/page.tsx) сейчас обёрнута в `<div className="mx-auto flex max-w-3xl flex-col gap-8 p-6">`. Переводим на структуру `/documents/[id]`: контент — **прямые дети грида** (хребет), маргиналии — `MarginNote`/`aside.margin-nav` как прямые дети грида. Хребет несёт: `LectureDetail`, экспорт, **селектор+рендер документа**, медиа-секцию (как есть, ссылки), offline/subscribe/share слоты, комментарии. Правое поле — колонка аннотаций активного документа.

`MarginNote` обязан быть прямым ребёнком `.page-grid` (грид в `<main>`, см. [[marginalia-layout-foundation]]) → структура страницы = фрагмент с контент-блоком и `MarginNote` как соседними детьми (как на странице документа), без центрирующей обёртки.

### 2. Селектор документа (вкладки-ссылки, `?doc=`)

Новый server-компонент-секция в слайсе `lectures` (напр. `LectureDocumentViewer`):
- Фетчит `getLectureDocuments(id, token)` — список видимых документов (по sort_order).
- Рендерит **строку ссылок** на документы: `href="?doc=<id>"` (сохраняя прочие query — `token` и т.п.), активная подсвечена (тот же акцентный стиль, что у активного таба: `--color-accent` подчёркивание + `--color-surface-subtle` заливка), `aria-current="page"` на активной. Это **навигация**, не ARIA-tablist (каждый документ = свой URL).
- **Активный документ** = `?doc=<id>`, если валиден (есть в списке видимых); иначе — **дефолт**: первый по sort_order. (СТОПГАП до бэкенд-флага; см. §«Бэкенд».)
- 0 документов → секция не рендерится (нет селектора, нет аннотаций).
- 1 документ → селектор-строку можно не показывать (рендерим единственный документ напрямую); аннотации работают так же.

### 3. Рендер активного документа (хребет)

- `getDocumentById(activeDoc, token)` → если `null` (приватный/удалён/нет доступа) — секция документа показывает фолбэк-сообщение (или дефолтит на первый доступный), но НЕ роняет всю страницу лекции.
- Тело — в `<div data-ast-root>` (субстрат движка) через `DocumentDetail` — ровно как на странице документа. На странице в DOM **ровно один** `data-ast-root` (активный документ) → инвариант движка (`querySelector("[data-ast-root]")`) соблюдён.

### 4. Аннотации (правое поле) — переиспользование как есть

- В `MarginNote side="end" grow` рендерим `<DocumentAnnotations parentId={activeDoc} />` ([document-annotations.tsx](../../../src/features/annotations/ui/document-annotations.tsx)) — server-компонент: фетч `getAnnotationsFor("document", activeDoc)` + RBAC + `loadSchemaIfNeeded` + `buildAnnotationCards` + `SchemaContextProvider` + client-коннектор `DocumentAnnotationLayer`. **Без изменений** — он доменно-агностичен к месту монтажа.
- Создание/редактирование/удаление — по `canCreateAnnotation`/`canEditAnnotation`/`canDeleteAnnotation` (как на странице документа). Анонимы видят аннотации (видимые), но не создают (нет capability).
- Колонка `MarginNotesColumn` позиционирует карточки **относительно своего контейнера** (`anchorRect.top − columnRect.top`) → корректно работает как поле грида; на узких — поток-список (фолбэк движка).

### 5. Ленивость по построению

Переключение документа = переход по ссылке `?doc=X` (Next client-side RSC-навигация). Сервер на каждый такой запрос грузит **только** активный документ (`getDocumentById`) и его аннотации (`getAnnotationsFor`) — остальные документы лекции НЕ фетчатся. Кеш-обёртки (`cache()`) держат дедуп в пределах запроса. Eager-фетч всех документов исключён.

### 6. Токен приватной лекции

Страница уже принимает `?token=` (share-link). Пробрасываем его в `getLectureDocuments(id, token)`, `getDocumentById(activeDoc, token)`, и сохраняем в ссылках селектора (`?doc=X&token=…`), чтобы приватный доступ не терялся при переключении.

## Поток данных

```
/lectures/[id]?doc=<D>&token=<T>  (server)
  ├─ getLectureById(id, T)                 → лекция (404 → notFound)
  ├─ getLectureDocuments(id, T)            → видимые документы (селектор + дефолт)
  ├─ active = ?doc валиден ? doc : первый-по-sort_order
  ├─ ХРЕБЕТ: LectureDetail, экспорт,
  │     <LectureDocumentViewer>            → строка-ссылок + <div data-ast-root><DocumentDetail active></div>
  │     медиа-секция (ссылки), комменты…
  └─ ПОЛЕ (MarginNote end): <DocumentAnnotations parentId={active}/>
                                            → DocumentAnnotationLayer цепляется к единственному data-ast-root
Переключение ?doc → навигация → сервер рендерит другой active (ленивость).
```

## Бэкенд (по правилу AGENTS.md «флаговать корень»)

1. **Видимость документов лекции (КРИТИЧНО для публичной страницы):** подтвердить, что `GET /api/lectures/{id}/documents` (и `GET /api/documents/{id}`) фильтруют/гейтят по видимости для зрителя (аноним/не-владелец) — приватные документы НЕ должны попасть на публичную страницу. Если фильтрации нет — это утечка, чинить на бэке. FE дополнительно прячет `getDocumentById → null`, но источник истины — бэк.
2. **Мейн-документ (`is_primary`) — уже открытый аск, скоро будет:** дефолтный `?doc` = мейн-документ. До флага — стопгап «первый по sort_order». Когда бэк добавит — поменять одну строку выбора дефолта (см. [[lecture-card-attachments-render]], тот же аск).

## Граничные случаи

- `?doc` отсутствует → дефолт (мейн-док / первый по sort_order).
- `?doc=<невалидный/невидимый>` → дефолт (не 404 всей страницы); опц. подсветить, что запрошенный недоступен.
- `getDocumentById(active) === null` → фолбэк-сообщение в секции документа, страница лекции жива.
- 0 документов → нет секции документов и аннотаций.
- 1 документ → без строки-селектора, рендер напрямую + аннотации.
- no-JS → ссылки `?doc=` работают (реальная навигация); активный документ + аннотации SSR-рендерятся; SSR-фолбэк коннектора даёт карточки списком.

## Вне области (YAGNI)

- Инлайн-рендер/плееры медиа на публичной странице — НЕ трогаем (остаётся секция-ссылки). Запрос только про документы.
- Admin-карточка `/admin/lectures/[id]` (клиентские lazy-табы `LectureDocumentTabs`) — НЕ трогаем.
- Мгновенный клиентский свитч документов на публичной (RSC-из-actions / клиентская пересборка карточек) — отклонено в пользу URL-driven (ленивость + простота + SSR).
- TOC активного документа в левом поле (как на `/documents/[id]`) — возможное последующее улучшение, не в этой итерации.

## Тестирование

- **Селектор/viewer:** дефолт = первый по sort_order; `?doc=<валид>` выбирает его; `?doc=<невалид>` → дефолт; активная ссылка несёт `aria-current` + акцентный стиль; токен сохраняется в ссылках.
- **Один root:** в DOM ровно один `data-ast-root` (активный документ).
- **Видимость:** приватный документ (не возвращённый списком / `getDocumentById→null`) не рендерится; страница жива.
- **0/1 документ:** секция скрыта / селектор скрыт.
- **SSR-фолбэк** аннотаций (карточки списком при `ready=false`) — присутствует.
- **Лейаут:** страница использует `.page-grid` (прямые грид-дети), аннотации в `MarginNote end`.

## Критерий готовности

Зелёные `pnpm lint && pnpm test && pnpm build`. Ручной браузер-приём: открыть лекцию с ≥2 документами; переключение `?doc=` грузит один документ + его аннотации; выделение текста → создание аннотации; аннотации совпадают с `/documents/[id]`; приватные документы не видны анониму; общий грид-лейаут + правое поле.
