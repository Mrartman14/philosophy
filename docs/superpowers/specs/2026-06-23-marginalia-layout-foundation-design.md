# Marginalia Layout Foundation — Design

**Дата:** 2026-06-23
**Статус:** дизайн одобрен, ожидает плана
**Тип:** foundation-PR (касается замороженных зон — см. §10)

## 1. Проблема и цель

Сейчас вёрстка центрирует контент через `<main>` шириной `max-w-screen-lg` (1024px) с
боковыми бордерами (`md:border-x`), а каждая страница внутри ставит свой ещё более узкий
контейнер (`mx-auto max-w-3xl`, 768px). Хедер дублирует 1024px + бордеры, образуя
непрерывную вертикальную линию.

Хотим:

1. Сохранить центрированный контент с максимальной шириной и непрерывные боковые бордеры
   + sticky-хедер.
2. Внутри центрированной колонки элементы занимают её **полную ширину** (не уже), обычный
   флоу сверху вниз — **без растяжки на всю свободную высоту**.
3. На десктопе иметь возможность размещать **page-specific** контент **слева и справа** от
   центрированной колонки — формат маргиналий / заметок на полях.
4. Нав-сайдбар (`/me`, `/admin`) — это основной контент, остаётся **во флоу**, а не в полях.
5. На мобильных — деградирует корректно.
6. Заложить устойчивый, консистентный с проектом **фундамент**, не доводя до пиксельной
   детализации каждый кейс.

### Решённые на брейншторме развилки

- **Природа контента полей:** гибкий фундамент — общий механизм «слотов в полях», который
  потом наполняется чем угодно (якорные сноски ИЛИ панели уровня страницы).
- **Геометрия:** узкий хребет (~720px) + поля **снаружи**; поля появляются только на широких
  экранах (≥1280px), ниже — скрыты/втекают в поток.
- **Хедер:** держит ширину хребта (~720px), бордеры продолжают линию хребта (макс.
  консистентность с текущей эстетикой «листа»).
- **Механизм инъекции:** глобальный именованный CSS-грид + kit-примитивы размещения
  (НЕ Next Parallel Routes, НЕ client-portal — см. §11 для обоснования).
- **RTL:** раскладка управляется логическими осями (`inline-start`/`inline-end`), зеркалится
  автоматически при `dir="rtl"`. Никаких ручных `dir`-условий в JS.

## 2. Модель трёх зон

Один full-width CSS-грид с **именованными линиями**. Имена логические, не `left`/`right`:

```
grid-template-columns:
  [bleed-start]   minmax(0, 1fr)
  [margin-start]  minmax(0, var(--layout-margin))
  [content-start] min(var(--layout-spine), 100%)
  [content-end]   minmax(0, var(--layout-margin))
  [margin-end]    minmax(0, 1fr)
  [bleed-end];
```

Центрирование хребта обеспечивают сами `minmax(0, 1fr)`-края (равные доли свободного места) — отдельный `justify-content: center` избыточен (no-op при гибких `1fr`-треках).

- **content** (хребет) — центрированная читаемая колонка.
- **margin-start / margin-end** — поля для маргиналий.
- **bleed** (1fr по краям + сами margin-треки в 0) — полноэкранный escape.

## 3. Токены ширины

Добавляются в источник дизайн-токенов (`tokens.generated.css` / `@theme`). Единицы — `rem`,
чтобы масштабироваться вместе с осью размера текста (`--text-scale` на `html`), консистентно с
текущими `max-w-*` (которые тоже rem-based).

| Токен | Дефолт | Смысл |
|---|---|---|
| `--layout-spine` | `45rem` (~720px) | ширина хребта (читаемая мера прозы); настраиваемая |
| `--layout-margin` | `14rem` (~224px) | ширина одного поля = ширине нав-сайдбара (`w-56`) |
| `--layout-gutter` | `2rem` | зазор хребет↔поле |

**Адаптив полей:** `--layout-margin` = `0` ниже `--breakpoint-xl` (1280px). Тогда на
ноутбуках/планшетах остаётся только центрированный 720-хребет — ровно нынешнее ощущение.
На `≥xl`: `720 + 2×(224 + 32) = 1232px < 1280px` — помещается с запасом.

## 4. Механизм размещения

Грид живёт в корневом `<main>`. **Дефолт:** `main > * { grid-column: content }` — любой прямой
потомок едет в хребет. → Существующие ~30 страниц со своим `mx-auto max-w-3xl`
**авто-работают** в 720-хребте без правок (обратная совместимость, см. §9).

Опт-ин в поля/bleed — через **kit-примитивы** (новые компоненты в `src/components/ui/`,
экспорт через `@/components/ui`). Чтобы работать на любой вложенности (а не только как прямой
потомок `main`), shell-обёртка использует **CSS subgrid**: элемент, растянутый на
`bleed-start / bleed-end` с `grid-template-columns: subgrid`, ре-экспонирует именованные
колонки своим детям.

Примитивы:

- **`<MarginNote side="start" | "end" collapse="inline" | "hidden">`** — рендерит `<aside>` в
  колонку поля. По умолчанию `collapse="inline"`: на `<xl` втекает в хребет под своим местом
  (инфа не теряется на мобиле). `"hidden"` — прячется на `<xl`.
- **`<FullBleed maxWidth?>`** — на весь viewport (`grid-column: bleed`). С `maxWidth` —
  широкий self-центрируемый контейнер (для app-страниц).
- **Дефолт** (без обёртки) — хребет.

Свойства механизма:

- **Полный SSR, ноль клиентского JS.** Контент полей попадает в initial HTML (важно для
  SEO / no-FOUC / no-JS — дух проекта).
- **RTL бесплатно:** `side="start|end"` логические; при `dir="rtl"` поля зеркалятся через
  поведение грида/логических свойств. Ручных `dir`-условий нет.
- **Единый источник истины:** один грид, именованные линии, токены ширины.

**Якорные сноски** (привязка к точке текста) фундамент поддерживает **позже без переделки**:
`<MarginNote>` ставится рядом с якорем в потоке, а вертикальное выравнивание добавляется
CSS/JS отдельной итерацией. В этом PR — только скелет зон, без выравнивания.

## 5. Хедер

`max-w-[var(--layout-spine)] mx-auto` + `border-x` → бордеры хедера совпадают с бордерами
хребта в одну непрерывную вертикаль (как сейчас, но 720 вместо 1024). Sticky, `z-50`,
backdrop-blur (`before:`-псевдоэлемент) — сохраняются.

Бордеры хребта рисует декоративный элемент (`aria-hidden`) c
`grid-column: content; grid-row: 1 / -1; border-inline: …` — непрерывная линия независимо от
высоты контента. `border-inline` (логическое) → корректно в RTL.

## 6. Два режима страниц

- **Reading (дефолт)** — 720-хребет + опциональные поля + хребет-бордеры. Для прозы:
  документы, лекции, тропы, глоссарий. Здесь живут маргиналии.
- **App/wide** — через `<FullBleed maxWidth=…>` (опц. удобная обёртка `<WideShell>` =
  FullBleed + `max-w` + паддинги): шире, без полей и без хребет-бордера. Для
  дашбордов/гридов/админки.

`/me` и `/admin`: нав-сайдбар остаётся «во флоу» (основной контент), но 720 для
«sidebar 224 + контент» тесно → секции переезжают на **app/wide** (FullBleed + свой `max-w` +
внутренний `flex`-сайдбар с логическим `border-e` сохраняется как есть). Маргиналий у них нет.
Визуальное следствие: эти секции теряют внешний хребет-бордер (приемлемо — у них свои
внутренние разделители). **Правило реализации:** декоративный `.spine-frame` обязан опт-аутиться
в app/wide-режиме — `.page-grid:has(> .col-bleed) .spine-frame { display: none }` (FullBleed/WideShell
эмитят `.col-bleed`), иначе фантомная 720-линия просвечивает за широким контентом.

## 7. Вертикальный флоу

Убираем растяжку контента на всю высоту. `body` сохраняет
`grid-rows-[var(--header-height)_1fr]`; `<main>` — это `1fr`-строка с `min-height` (фон/футер
добивают вьюпорт), но контент пакуется к верху: `align-content: start`. Снимаем `items-center`
со старого main (центрирование теперь делают `1fr`-края грида + `content`-колонка). Контент
течёт сверху вниз естественной высоты.

## 8. Адаптив (сводка)

| Брейкпоинт | Поведение |
|---|---|
| `< md` (768) | хребет = 100vw, без бордеров (как сейчас); `MarginNote` втекает в поток (`inline`) или скрыт |
| `md … xl` (768–1280) | хребет 720 центрирован + бордеры; полей нет; `MarginNote` втекает/скрыт |
| `≥ xl` (1280) | хребет 720 + поля по бокам (`--layout-margin` раскрывается) |

## 9. Обратная совместимость и охват PR

Охват foundation-PR:

1. Токены ширины (`--layout-spine`, `--layout-margin`, `--layout-gutter`).
2. Грид в корневом `<main>` + декоративный хребет-бордер + `align-content: start`.
3. Три kit-примитива: `MarginNote`, `FullBleed`, (опц.) `WideShell`. Экспорт из
   `@/components/ui`.
4. Миграция: `src/app/layout.tsx` (main), хедер (на ширину хребта), шеллы
   `src/app/me/layout.tsx` + `src/app/admin/layout.tsx` (на app/wide).
5. ~30 контент-страниц со своим `mx-auto max-w-3xl` — **массово НЕ трогаем**: авто-работают в
   хребте (дефолт-колонка). Внутренний `max-w-3xl` снимается постранично позже (контент займёт
   полный хребет). 1–2 прозовые страницы (документ/лекция) мигрируем как референс + ставим
   демонстрационную `MarginNote`.
6. Тесты: грид-раскладка, поведение примитивов (`side`, `collapse`), RTL-зеркалирование,
   адаптив (поля раскрываются на `≥xl`). Существующие layout-снапшоты обновляются.
7. Перед мержем зелёные: `pnpm lint && pnpm test && pnpm build`.

## 10. Замороженные зоны (foundation-контекст)

Согласно `AGENTS.md`, касание этих зон = отдельный foundation-PR (это он и есть, легитимно):

- `src/app/layout.tsx` — root shell.
- `src/app/globals.css` + `src/styles/tokens.generated.css` — глобальные стили/токены.
- `src/components/ui/*` — новые kit-примитивы (этот PR — место для их обсуждения).
- `src/app/me/layout.tsx`, `src/app/admin/layout.tsx` — секционные шеллы.

ESLint-гарды (kit-only / логические свойства / RTL) — соблюсти.

## 11. Отклонённые альтернативы

- **Next.js Parallel Routes (`@marginStart`/`@marginEnd`):** натив, независимый стриминг
  слотов. Отклонено: тройное дублирование дерева роутов, `default.tsx` повсюду, жёсткая
  привязка к URL, неудобно для будущих якорных сносок.
- **Client context/portal (`<Marginalia>` телепортирует через React-портал):** ставится где
  угодно (рядом с якорем). Отклонено: portals — client-only, контент поля не попадает в
  SSR-HTML (FOUC, плохо для no-JS/SEO) — против SSR-first духа проекта.

## 12. Открытые настраиваемые параметры (не блокеры)

- Точное число `--layout-spine` (дефолт `45rem`/720px) — тюнится токеном.
- Ширина/зазор полей (`14rem` / `2rem`).
- Брейкпоинт раскрытия полей (`xl`/1280px).
- Дефолт `collapse` у `MarginNote` (`inline`).
