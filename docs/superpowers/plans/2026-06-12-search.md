# План реализации фичи `search` (глобальный поиск)

> **Для агентов-исполнителей:** ОБЯЗАТЕЛЬНЫЙ САБ-СКИЛЛ: используй
> superpowers:subagent-driven-development (рекомендуется) или
> superpowers:executing-plans для пошагового выполнения. Шаги размечены
> чекбоксами (`- [ ]`).

**Goal:** Слайс `src/features/search/` + публичная страница `/search` —
глобальный поиск по лекциям и терминам глоссария через `GET /api/search`
с фильтром по типу, локальной пагинацией (паттерн audit) и прямой ссылкой
на `.md`-выгрузку результатов; плюс foundation-touch — восстановление
`SearchInput` в шапке (там сейчас TODO).

**Architecture:** Read-only фича по образцу `src/features/audit/`. Состояние
запроса живёт в URL `searchParams` (`?q=&type=&offset=`); страница —
server component, парсит параметры Zod-схемой (`.catch(undefined)` —
битый URL не валит страницу), зовёт серверный fetcher `getSearchResults`,
рендерит разнотипные хиты (`type: "lecture" | "glossary"`) как ссылки на
`/lectures/[id]` и `/glossary/[id]`. Пагинация локальная
(паттерн `audit-pagination`, мержит `offset` в текущий query, сохраняя
`q`/`type`). Мутаций нет → `actions.ts`, `permissions.ts`, тег в
`src/api/tags.ts` не нужны (осознанное отклонение, как у audit). Эндпоинт
публичный (`optionalAuth` + rate-limit), видимость скоупится бекендом по
актору — гейтить страницу/capability НЕ надо.

**Tech Stack:** Next.js 16 (App Router, RSC + server-only), TypeScript
(`exactOptionalPropertyTypes`), Zod, openapi-fetch (`createApiClient`),
Vitest, Tailwind v4 (CSS-переменные), Base UI (через `@/components/ui`).

## Контекст из бекенда (источник истины — `philosophy-api`)

Сверено по коду, не по комментариям (правило спеки §6.14):

- **Роут:** `GET /api/search` →
  `publicRL.Middleware(optionalAuth(...))` (`cmd/server/main.go:990`).
  **Публичный, optionalAuth, rate-limited.** Без `shareTokenMW`.
  Анонимный актор не получает приватный контент — фильтрация в SQL по
  `Actor` из контекста. ⇒ **гейт страницы и capability НЕ нужны.**
- **Параметры запроса** (`internal/search/handler.go:34-70`):
  - `q` — **обязателен**, max **200** символов (иначе 422 / поле `q`).
  - `type` — опционально, enum строго `"lecture" | "glossary"`
    (`validTypes`, handler.go:11-14); иное → 422 (поле `type`).
  - пагинация: `ParsePagination(r, 20, 100)` — **default limit 20,
    max 100**, `offset` от 0.
- **Форма ответа** (`internal/search/model.go`,
  schema.ts `components["schemas"]["search.Hit"]`):
  - конверт `httputil.ListResponse` → `{ data: search.Hit[],
    pagination: { offset, limit, total } }`.
  - `search.Hit`: `{ type?: string; lecture?: search.LectureData;
    glossary_term?: search.GlossaryData; matches?: search.Match[] }`.
  - `search.LectureData`: `{ lecture_id?; title?; date? }`.
  - `search.GlossaryData`: `{ id?; title?; definition? }`.
  - `search.Match`: `{ source?; snippet?; block_id?; block_type? }`.
  - **Только два типа хитов**: lecture и glossary. Документы/медиа/
    комментарии в выдачу НЕ индексируются (вопреки формулировке скоупа
    §3 «document→…» — на беке таких источников нет:
    `NewSearcher(NewGlossarySource, NewLectureSource)`, main.go:486-489).
    План рендерит только эти два типа; неизвестный `type` —
    graceful-skip (defensive).
- **Ссылки на детали** (как делает `.md`-handler,
  `handler_md.go:109-121`): lecture → по `lecture_id`,
  glossary → по `GlossaryData.id`. На фронте:
  `/lectures/${lecture_id}`, `/glossary/${id}` (роуты существуют:
  `src/app/lectures/[id]/page.tsx`, `src/app/glossary/[id]/page.tsx`).
- **`.md`-выгрузка:** `GET /api/search.md` — те же параметры
  (`q`, `type`, `limit`, `offset`), публичная (main.go:1004). Фронт даёт
  **прямую ссылку** на бек (паттерн `glossaryExportUrls` /
  `GlossaryExportLinks`) — контент рендерит бек.

## Уроки волн 1–2 (применять во всех тасках)

- **`exactOptionalPropertyTypes`**: опциональные поля типизируй как
  `?: T | undefined`; при сборке query/filter-объектов не присваивай
  `undefined` напрямую — используй условный спред
  (`...(x ? { x } : {})`), как в `audit/api.ts` и `audit/page.tsx`.
- **`server-only`**: `api.ts`, `schemas.ts` начинаются с
  `import "server-only";`. `export-urls.ts` — чистый helper БЕЗ
  `server-only` (нужен тестам; паттерн `glossary/export-urls.ts`).
- **Локальная пагинация**: ui-kit `Pagination` теряет прочие searchParams —
  пишем слайс-локальный `search-pagination` (копия логики
  `audit-pagination`, мерж offset в текущий query).
- **Фильтры через URL**: client-компонент обновляет URL через
  `router.replace()`/`router.push()`; revalidation бесплатна, fetcher
  перезапускается (конвенции §3.5).
- **Типы из `@/api/schema`** (не из несуществующего `@/api/types` —
  старая фича его импортила, его больше нет).

import "server-only";
import { z } from "zod";
import { SEARCH_TYPES } from "./types";

/**
 * Валидация параметров поиска из URL searchParams.
 *
 * Строгие суб-схемы экспортируются для прямого unit-тестирования
 * failure-кейсов. В композитной SearchParamsSchema каждое поле обёрнуто
 * в .optional().catch(undefined): битый параметр (рукописный URL,
 * устаревшая закладка) молча отбрасывается, страница не падает.
 *
 * Лимиты сверены с беком (internal/search/handler.go):
 *  - q: required, max 200 символов;
 *  - type: enum lecture|glossary;
 *  - offset: >= 0. (limit фиксируем на фронте, в URL не выносим.)
 */

export const SearchQuerySchema = z
  .string()
  .trim()
  .min(1, "Введите запрос")
  .max(200, "Не более 200 символов");

export const SearchTypeSchema = z.enum(SEARCH_TYPES);

export const SearchOffsetSchema = z.coerce.number().int().min(0);

export const SearchParamsSchema = z.object({
  q: SearchQuerySchema.optional().catch(undefined),
  type: SearchTypeSchema.optional().catch(undefined),
  offset: SearchOffsetSchema.optional().catch(undefined),
});

export type SearchParamsInput = z.infer<typeof SearchParamsSchema>;
```

// src/features/search/api.ts
import "server-only";
import { cache } from "react";
import { createApiClient } from "@/api/client";
import type { SearchHit, SearchType } from "./types";

export interface SearchFilter {
  /** Поисковый запрос. Уже тримленый/валидный (см. schemas.ts). */
  q: string;
  /** Сужение по типу источника. undefined = все (lecture + glossary). */
  type?: SearchType | undefined;
  offset?: number | undefined;
  /** Бек: default 20, max 100. Фронт фиксирует на странице. */
  limit?: number | undefined;
}

export interface SearchResult {
  items: SearchHit[];
  total: number;
  offset: number;
  limit: number;
}

/**
 * Глобальный поиск. Read-only фича: мутаций нет, revalidateEntity не нужен,
 * тег в src/api/tags.ts не заводится. unstable_cache не используется —
 * выдача зависит от актора (видимость скоупится беком) и от полнотекстового
 * запроса; React.cache дедуплицирует вызовы в рамках одного запроса.
 *
 * Эндпоинт публичный (optionalAuth) — createApiClient приложит JWT из cookie,
 * если он есть (авторизованный видит свой приватный контент), иначе аноним.
 */
export const getSearchResults = cache(
  async (filter: SearchFilter): Promise<SearchResult> => {
    const api = await createApiClient();
    const offset = filter.offset ?? 0;
    const limit = filter.limit ?? 20;
    const query: {
      q: string;
      offset: number;
      limit: number;
      type?: string;
    } = { q: filter.q, offset, limit };
    if (filter.type) query.type = filter.type;

    const { data, error } = await api.GET("/api/search", {
      params: { query },
    });
    if (error) {
      throw new Error(error.error ?? "Не удалось выполнить поиск");
    }
    return {
      items: (data?.data ?? []) as SearchHit[],
      total: data?.pagination?.total ?? 0,
      offset: data?.pagination?.offset ?? offset,
      limit: data?.pagination?.limit ?? limit,
    };
  },
);
```

// Чистый helper построения ссылки на .md-выгрузку результатов поиска.
// Без "server-only": нужен тестам; никаких side effects и зависимостей
// (паттерн — src/features/glossary/export-urls.ts).
import type { SearchType } from "./types";

export interface SearchExportParams {
  q: string;
  type?: SearchType | undefined;

/**
 * Строит абсолютный URL публичной .md-выгрузки поиска (контент рендерит бек):
 * GET /api/search.md?q=…[&type=…]. Эндпоинт ПУБЛИЧНЫЙ (cmd/server/main.go:1004

 *
 * limit/offset в ссылку НЕ выносим: выгрузка отдаёт первую страницу
 * результата по дефолту бека (20), как осознанный MVP-компромисс.
 */
export function searchExportMdUrl(
  apiBase: string,
  params: SearchExportParams,
): string {

  const qs = new URLSearchParams({ q: params.q });
  if (params.type) qs.set("type", params.type);
  return `${base}/api/search.md?${qs.toString()}`;
}

import type { SearchHit, SearchMatch } from "../types";

const TYPE_LABEL: Record<string, string> = {
  lecture: "Лекция",
  glossary: "Термин",
};

const dateFormat = new Intl.DateTimeFormat("ru-RU", { dateStyle: "long" });

function formatDate(iso?: string): string | null {
  if (!iso) return null;
  const d = new Date(iso);
  if (Number.isNaN(d.getTime())) return null;
  return dateFormat.format(d);
}

/** Первые до 3 непустых сниппетов хита (бек уже усекает текст). */
function topSnippets(matches: SearchMatch[] | undefined): string[] {
  if (!matches) return [];
  const out: string[] = [];
  for (const m of matches) {
    const s = m.snippet?.trim();
    if (s) out.push(s);
    if (out.length >= 3) break;
  }
  return out;
}

function hitHref(hit: SearchHit): string | null {
  if (hit.type === "lecture" && hit.lecture?.lecture_id) {
    return `/lectures/${hit.lecture.lecture_id}`;
  }
  if (hit.type === "glossary" && hit.glossary_term?.id) {
    return `/glossary/${hit.glossary_term.id}`;
  }
  return null;
}

function hitTitle(hit: SearchHit): string {
  if (hit.type === "lecture") return hit.lecture?.title?.trim() || "Без названия";
  if (hit.type === "glossary") return hit.glossary_term?.title?.trim() || "Без названия";
  return "Результат";
}

function HitCard({ hit }: { hit: SearchHit }) {
  const href = hitHref(hit);
  // Неизвестный тип или хит без id — нечего открыть, пропускаем (defensive,
  // как .md-handler бека: unknown hit types skipped).
  if (!href) return null;

  const title = hitTitle(hit);
  const label = hit.type ? (TYPE_LABEL[hit.type] ?? hit.type) : "—";
  const date = hit.type === "lecture" ? formatDate(hit.lecture?.date) : null;
  const snippets = topSnippets(hit.matches);

  return (
    <article className="rounded-lg border border-(--color-border) bg-(--color-background) p-4">
      <header className="flex flex-wrap items-baseline gap-2">
        <span className="rounded bg-(--color-text-pane) px-2 py-0.5 text-xs text-(--color-description)">
          {label}
        </span>
        <Link href={href} className="text-lg font-semibold hover:underline">
          {title}
        </Link>
        {date && (
          <time className="text-xs text-(--color-description)">{date}</time>
        )}
      </header>
      {snippets.length > 0 && (
        <ul className="mt-2 flex flex-col gap-1">
          {snippets.map((s, i) => (
            <li
              key={i}
              className="rounded px-2 py-1 text-sm text-(--color-text) hover:bg-(--color-text-pane)"
            >
              {s}
            </li>
          ))}
        </ul>
      )}
    </article>
  );
}

interface Props {
  hits: SearchHit[];
  total: number;
}

export function SearchResults({ hits, total }: Props) {
  if (hits.length === 0) {
    return (
      <EmptyState
        title="Ничего не найдено"
        description="Попробуйте изменить запрос или снять фильтр по типу."

      <p className="text-xs text-(--color-description)">Найдено: {total}</p>
      <ul className="flex flex-col gap-3">
        {hits.map((hit, i) => (
          <li key={hitHref(hit) ?? i}>
            <HitCard hit={hit} />
          </li>
        ))}
      </ul>
    </div>
  );
}

"use client";
// src/features/search/ui/search-pagination.tsx
import Link from "next/link";
import { usePathname, useSearchParams } from "next/navigation";

interface Props {
  offset: number;
  limit: number;
  total: number;
}

const linkCls =
  "rounded border border-(--color-border) px-3 py-1 hover:bg-(--color-text-pane)";
const disabledCls =
  "rounded border border-(--color-border) px-3 py-1 opacity-40";

/**
 * Слайс-локальная пагинация. ui-kit `Pagination` строит href только из
 * offset и теряет q/type; src/components/ui — запретная зона. Этот компонент
 * мержит offset в ТЕКУЩИЙ query — q и type выживают.
 */
export function SearchPagination({ offset, limit, total }: Props) {
  const pathname = usePathname();
  const searchParams = useSearchParams();

  function makeHref(nextOffset: number): string {
    const params = new URLSearchParams(searchParams.toString());
    if (nextOffset > 0) params.set("offset", String(nextOffset));
    else params.delete("offset");
    const qs = params.toString();
    return qs ? `${pathname}?${qs}` : pathname;
  }

  const hasPrev = offset > 0;
  const hasNext = offset + limit < total;

  return (
    <nav aria-label="Пагинация" className="flex items-center gap-2 text-sm">
      {hasPrev ? (
        <Link href={makeHref(Math.max(0, offset - limit))} className={linkCls}>
          ← Назад
        </Link>
      ) : (
        <span className={disabledCls}>← Назад</span>
      )}
      <span className="text-(--color-description)">
        {total === 0
          ? "0 из 0"
          : `${offset + 1}–${Math.min(offset + limit, total)} из ${total}`}
      </span>
      {hasNext ? (
        <Link href={makeHref(offset + limit)} className={linkCls}>
          Вперёд →
        </Link>
      ) : (
        <span className={disabledCls}>Вперёд →</span>
      )}
    </nav>
  );
}

## Риски и допущения

1. **Форма результата и типы сущностей (главное).** Бек индексирует
   **только два типа** — `lecture` и `glossary`
   (`internal/search/handler.go validTypes`; `NewSearcher(GlossarySource,
   LectureSource)`, main.go:486-489). Формулировка скоупа «document→…,
   и т.д.» опережает бек — документы/медиа/комментарии в глобальном поиске
   НЕ участвуют. План рендерит ровно lecture+glossary; неизвестный `type` —
   graceful-skip (как `.md`-handler бека). Если бек позже добавит источник —
   расширить `SEARCH_TYPES`, `TYPE_LABEL`, `hitHref`/`hitTitle` в
   `search-results.tsx` и `SearchTypeSchema` (через `types.ts`).
2. **`search.Hit` — плоский разнотипный конверт.** `type` отличает,
   `lecture`/`glossary_term` — взаимоисключающие nullable-объекты, `matches`
   — сниппеты. Это НЕ старая форма (`SearchLectureHit`/`segment_id`/
   `match.text` из удалённой фичи `daf21dd1`) — её не воскрешать.
3. **Публичность эндпоинта.** Сверено по `main.go:990`: optionalAuth +
   rate-limit, без shareToken. Гейта/`can*`/capability НЕТ. Если бек
   когда-нибудь закроет search под auth — добавить
   `const me = await getMe(); if (!me) redirect("/login")` в странице
   (сейчас не нужно).
4. **`.md`-выгрузка отдаёт первую страницу.** `searchExportMdUrl` не несёт
   `offset`/`limit` — выгрузка покажет дефолтные 20 результатов бека, а не
   текущую страницу UI. Осознанный MVP-компромисс (как у glossary list md).
   При необходимости — добавить `offset`/`limit` в `SearchExportParams`.
5. **Видимость скоупится беком по актору.** `createApiClient` приложит JWT
   из cookie: авторизованный увидит свой приватный контент, аноним — только
   публичный. Фронт ничего не фильтрует — доверяет беку (правило §6).
6. **`q` max 200 — двойная защита.** Схема режет на парсинге (битый URL →
   `q` undefined → «введите запрос»); `maxLength={200}` на инпутах не даёт
   ввести лишнего. Бек — третья линия (422). Лимит сверен с handler.go:40.
7. **Иконка поиска.** Зависимость от `@/assets/icons/search-icon` —
   условная (Task 7 Step 2): если файла нет, используется inline-SVG в
   самом компоненте. UI-kit (`src/components/ui`) не трогаем — запретная
   зона.
8. **`SearchBody` как async-подкомпонент страницы.** Вынесен ради
   `try/catch` вокруг fetcher (бек недоступен → дружелюбное сообщение, не
   500). Альтернатива — `error.tsx`, но локальный fallback ближе к UX
   старой фичи и не роняет форму поиска.
9. **Параллельная безопасность.** Единственный shared-файл —
   `app-header.tsx` — изолирован в Task 9 (foundation-touch после мержа).
   `src/api/tags.ts` НЕ трогаем (read-only, тега нет). Прочие планы волны 3
   в `src/features/search/` и `src/app/search/` не пишут.
