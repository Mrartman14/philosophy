> **Правила параллельной работы (передавай дословно каждому субагенту):** никаких `git stash` / `git reset` / `git checkout .` / `git clean`; `git add` только своих файлов по имени (НЕ `git add -A` / `git add .`); не откатывать и не перезаписывать чужие изменения; передавать эти правила своим субагентам.

**Goal:** Реализовать слайс `src/features/comments/` — дерево аргументационных комментариев на странице лекции с типами узлов, пер-осевыми реакциями, созданием/редактированием через AST-редактор, поиском, ревизиями, subtree-просмотром, .md/.txt-ссылками и пер-лекционной admin-модерацией.

**Architecture:** SSR-first слайс по конвенциям `docs/frontend-conventions.md`. Серверные fetchers (`api.ts`, `React.cache` + `unstable_cache` с тегом `comments`), server actions (`createAction`/`createFormAction` + `requireCapability` + `revalidateEntity`), доменные `can*`-хелперы (`permissions.ts`), Zod-схемы (`schemas.ts`). UI: дерево комментариев композируется страницей `src/app/lectures/[id]/page.tsx` (единственный владелец этого файла в волне 2 — мы); admin-модерация — `/admin/comments`; subtree — `/comments/[id]`. Создание/редактирование комментариев — через готовый `<AstEditor entityContext="comment">`. Реакции — client-компоненты, вызывающие server actions (toggle через POST upsert / DELETE оси). Матрица типов и допустимых осей берётся из `GET /api/comments/schema` (фетчится на сервере, кешируется).

**Tech Stack:** Next.js 16 (App Router, RSC + server actions), TypeScript (`exactOptionalPropertyTypes`), Zod, openapi-fetch (типы из `src/api/schema.ts`), Vitest + jsdom, готовые компоненты `@/components/ast-editor`, `@/components/ast-render`, `@/components/revision-history`, UI-kit `@/components/ui`.

| Тип | Семантика | Можно корнем? | Допустимые оси реакций |
| --- | --- | --- | --- |

Матрица «какой тип допустим под каким родителем» и «какие оси допустимы для типа» — НЕ хардкодим в UI: берём из `GET /api/comments/schema` (поля `allowed_roots`, `allowed_children`, `allowed_reactions`, `max_depth`). Бек всё равно провалидирует и вернёт 422 (`INVALID_ROOT_TYPE`, `INVALID_TYPE_FOR_PARENT`, `MAX_DEPTH_EXCEEDED`, `AXIS_NOT_ALLOWED`) — UI лишь предотвращает заведомо неверный выбор.

### Реакции (пер-осевые тоггл)

- Оси: `agreement` (±1), `quality` (±1), `insight` (только +1).
- Поставить: `POST /api/comments/{id}/reactions` body `{ axis, value }` (upsert; 204).
- Снять ось: `DELETE /api/comments/{id}/reactions/{axis}` (204).
- Нельзя реагировать на свой комментарий → 403 `SELF_REACTION`.
- Чужая ось для типа → 422 `AXIS_NOT_ALLOWED`.
- `insight` со значением ≠ 1 → 400 `INVALID_INSIGHT_VALUE`.
- На удалённый (tombstone) комментарий → 422 `COMMENT_DELETED`.
- Сводка в `Comment.reactions` (`{agreement:{positive,negative}, quality:{positive,negative}, insight:number}`); моя реакция в `Comment.my_reactions` (`{agreement?:number, quality?:number, insight?:boolean}`).

### Создание / редактирование / удаление

- Создать: `POST /api/lectures/{id}/comments` (гейт `comment.create` — есть и у user, и у admin). Body `comment.CreateRequest` `{ type, blocks (min 1, max 200), parent_id?, anchor? }`. Ответ 201 → `comment.Comment`.
- Редактировать blocks: `PUT /api/comments/{id}/blocks` (bulk-replace, **owner-only** — `c.UserID == me.id`, без admin-override). Body `{ blocks }`. Коды: 422 `BLOCKS_EMPTY` / `BLOCKS_INVALID` / `BLOCK_ID_UNKNOWN` / `DUPLICATE_BLOCK_ID` / `REF_NOT_FOUND` / `COMMENT_DELETED`; 409 `BLOCKS_HAVE_ANCHORS` / `BLOCK_REFERENCED`.
- Удалить: `DELETE /api/comments/{id}` (owner-only) ИЛИ `DELETE /api/admin/comments/{id}` (гейт `comment.delete_any` — **действует независимо от видимости/периметра лекции**, спец-правило для comments из §6.2 спеки). Коды: 409 `COMMENT_REFERENCED` / `BLOCKS_HAVE_ANCHORS`; 422 `COMMENT_DELETED`. Бек сам выбирает hard- (лист) или soft-delete (tombstone, если есть дети).
- Удалённый узел приходит как tombstone: `is_deleted: true`, `blocks: []`, `author/user_id/anchor/reactions` обнулены — рендерим заглушкой «Комментарий удалён», но детей показываем.

### Чтение

- Список корней лекции с поддеревьями: `GET /api/lectures/{id}/comments` (public, optionalAuth) → `[]comment.RootSubtree` (`{root, descendants[]}`), пагинация по корням (limit default 20 / max 100), опц. `?block_id=`. `my_reactions` приходят только если есть auth.
- Поддерево узла: `GET /api/comments/{id}/subtree` (public) → `comment.RootSubtree`.
- Поиск: `GET /api/lectures/{id}/comments/search?q=` (**requiredAuth**!) → `[]comment.CommentSummary` (`{id, type, snippet, author, created_at, ...}`). Аноним получит 401 → кнопку поиска показываем только залогиненным; гейт периметра лекции на беке (404 если не видит лекцию).
- Схема: `GET /api/comments/schema` (public).
- Резолв якоря: `GET /api/blocks/{block_id}` (public, optionalAuth) → `httputil.Response` (data НЕ типизирован — типизируем вручную как `ast.Block`).
- Ревизии: `GET /api/comments/{id}/revisions` (public) → `[]revision.RevisionMeta`; `GET /api/comments/{id}/revisions/{revisionID}` → `revision.Revision`. Бек отдаёт ASC (старые первыми) — переворачиваем перед `RevisionHistory`.

### .md/.txt выгрузки — ПУБЛИЧНЫЕ (прямые `<a href>`, НЕ прокси)

Проверено по `cmd/server/main.go:995-998`: эти роуты обёрнуты только `publicRL.Middleware` без auth — токен не нужен. В отличие от events (там был прокси), здесь ссылки прямые, как у glossary:

- `GET {API_URL}/api/lectures/{id}/comments.md` и `.txt`
- `GET {API_URL}/api/comments/{id}/subtree.md` и `.txt`

Ссылка ведёт прямо на бек: `${process.env.NEXT_PUBLIC_API_URL ?? ""}` нельзя — на клиенте нет `API_URL`. Используем относительный путь к беку через публичную базу. **ВАЖНО:** клиентский код не знает `API_URL` (он server-only в `me.ts`/`client.ts`). Поэтому ссылки строим как абсолютные к API через переменную окружения, доступную клиенту, либо как относительный путь, проксируемый Next. Решение для этого плана: ссылки рендерим server-компонентом, который читает `process.env.API_URL` на сервере и кладёт готовый href в `<a>` (server component может читать env). См. задачу 9 — там это зафиксировано конкретным кодом.

### Anchor (модель якоря)

`comment.Anchor` = `{ target_entity_type: "document"|"glossary"|"comment"|"media", target_entity_id, + поля anchor.Position: start_block_id, start_char, end_block_id, end_char, exact, prefix, suffix, start_sec, end_sec }`. В MVP-скоупе этой фичи **создание якоря через UI не делаем** (это сложный selection-flow; в спеке §3 для comments якорь упомянут только как «резолв якорей для контекста» через `GET /api/blocks/{block_id}`). Мы:
- читаем `comment.anchor` из ответа и, если он есть и `target_entity_type` резолвится, показываем контекстную плашку «Привязка к фрагменту» с резолвом блока через `GET /api/blocks/{block_id}` (для text-anchor) — задача 13;
- НЕ строим UI выбора якоря при создании (передаём `anchor` отсутствующим). Это зафиксированное упрощение — см. «Риски / допущения».

// src/features/comments/permissions.ts
import "server-only";
import type { MaybeMe } from "@/utils/me";
import { can, isMutationAllowed } from "@/utils/permissions";
import type { Comment } from "./types";

/**
 * Создание комментария: гейт `comment.create` (есть у user И admin).
 *
 * ВНИМАНИЕ: `comment.create` пока НЕ входит в union `Capability`
 * (src/utils/permissions.ts — запретная зона). Поэтому чек делаем локально:
 * active-пользователь + членство в capabilities. Это паттерн волны 1 для
 * cap'ов, ещё не внесённых в union.
 * TODO(foundation-update): добавить "comment.create" в union Capability и
 * заменить на can(me, "comment.create").
 */
export function canCreateComment(me: MaybeMe): boolean {
  if (!isMutationAllowed(me)) return false;
  return me.capabilities.includes("comment.create");
}

/**
 * Редактирование blocks — owner-only, без admin-override
 * (бек: service.go Update → c.UserID == me.id). Удалённый — нельзя.
 */
export function canEditComment(
  me: MaybeMe,
  comment: Pick<Comment, "user_id" | "is_deleted">,
): boolean {
  if (!isMutationAllowed(me)) return false;
  if (comment.is_deleted) return false;
  return comment.user_id === me.id;
}

/**
 * Удаление — owner ИЛИ admin с `comment.delete_any`.
 * Для comments delete_any действует НЕЗАВИСИМО от видимости/периметра лекции
 * (спека §6.2). Уже удалённый — нельзя (бек: 422 COMMENT_DELETED).
 */
export function canDeleteComment(
  me: MaybeMe,
  comment: Pick<Comment, "user_id" | "is_deleted">,
): boolean {
  if (!isMutationAllowed(me)) return false;
  if (comment.is_deleted) return false;
  if (comment.user_id === me.id) return true;
  return can(me, "comment.delete_any");
}

/**
 * Реакция — любой active-пользователь на ЧУЖОЙ не-удалённый комментарий.
 * Свой → 403 SELF_REACTION (бек), поэтому прячем кнопки на своих.
 */
export function canReactToComment(
  me: MaybeMe,
  comment: Pick<Comment, "user_id" | "is_deleted">,
): boolean {
  if (!isMutationAllowed(me)) return false;
  if (comment.is_deleted) return false;
  return comment.user_id !== me.id;
}

/** Поиск по комментариям лекции требует auth (бек: requiredAuth). */
export function canSearchComments(me: MaybeMe): boolean {
  return isMutationAllowed(me);
}

/** Admin-модерация (GET /api/admin/comments, admin delete) — comment.delete_any. */
export function canModerateComments(me: MaybeMe): boolean {
  return can(me, "comment.delete_any");
}

// src/features/comments/schemas.ts
import "server-only";
import { z } from "zod";

const COMMENT_TYPES = [
  "claim",
  "grounds",
  "rebuttal",
  "qualifier",
  "question",
  "answer",
  "offtop",
  "summary",
] as const;

const AXES = ["agreement", "quality", "insight"] as const;

/**
 * blocks приходит из скрытого input формы как JSON-строка (AstEditor
 * сериализует blocks в hidden input). Парсим в непустой массив.
 * Пустой массив → 422 BLOCKS_EMPTY на беке; ловим раньше.
 */
const BlocksJsonSchema = z
  .string()
  .min(1, "Тело не может быть пустым")
  .transform((s, ctx) => {
    try {
      const parsed: unknown = JSON.parse(s);
      if (!Array.isArray(parsed) || parsed.length === 0) {
        ctx.addIssue({
          code: z.ZodIssueCode.custom,
          message: "Комментарий не может быть пустым",
        });
        return z.NEVER;
      }
      return parsed as unknown[];
    } catch {
      ctx.addIssue({ code: z.ZodIssueCode.custom, message: "Битый JSON в теле" });
      return z.NEVER;
    }
  });

export const CommentCreateSchema = z
  .object({
    type: z.enum(COMMENT_TYPES, { message: "Неизвестный тип комментария" }),
    blocks: BlocksJsonSchema,
    // parent_id есть только в форме ответа; в корневой форме отсутствует.
    parent_id: z.string().uuid("Некорректный parent_id").optional(),
  })
  .transform((raw) => ({
    type: raw.type,
    blocks: raw.blocks,
    ...(raw.parent_id ? { parent_id: raw.parent_id } : {}),
  }));

export const CommentBlocksUpdateSchema = z.object({
  id: z.string().uuid("Некорректный id комментария"),
  blocks: BlocksJsonSchema,
});

export const ReactionSchema = z
  .object({
    id: z.string().uuid("Некорректный id комментария"),
    axis: z.enum(AXES, { message: "Неизвестная ось реакции" }),
    // value приходит строкой из action-аргумента/формы.
    value: z.coerce.number().int(),
  })
  .superRefine((v, ctx) => {
    if (v.value !== 1 && v.value !== -1) {
      ctx.addIssue({ code: z.ZodIssueCode.custom, path: ["value"], message: "Значение должно быть +1 или -1" });
    }
    if (v.axis === "insight" && v.value !== 1) {
      ctx.addIssue({ code: z.ZodIssueCode.custom, path: ["value"], message: "insight допускает только +1" });
    }
  });

export const RemoveReactionSchema = z.object({
  id: z.string().uuid("Некорректный id комментария"),
  axis: z.enum(AXES, { message: "Неизвестная ось реакции" }),
});

export const CommentIdSchema = z.object({
  id: z.string().uuid("Некорректный id комментария"),
});

export type CommentCreateInput = z.infer<typeof CommentCreateSchema>;
export type CommentBlocksUpdateInput = z.infer<typeof CommentBlocksUpdateSchema>;
export type ReactionInput = z.infer<typeof ReactionSchema>;
export type RemoveReactionInput = z.infer<typeof RemoveReactionSchema>;
export type CommentIdInput = z.infer<typeof CommentIdSchema>;

"use client";
// src/features/comments/ui/comment-reactions.tsx
import { useState, useTransition } from "react";
import { setReaction, removeReaction } from "../actions";
import { axisLabel, REACTION_AXES } from "../reactions";
import type { MyReactions, ReactionAxis, ReactionSummary } from "../types";

interface Props {
  commentId: string;
  type: import("../types").CommentType;
  reactions: ReactionSummary | undefined;
  myReactions: MyReactions | undefined;
  /** Допустимые оси для этого типа (из schema.allowed_reactions[type]). */
  allowedAxes: ReactionAxis[];
  /** Можно ли вообще реагировать (active, не свой, не удалённый). */
  canReact: boolean;
}

/** Текущее значение моей реакции по оси (number для agreement/quality, 1|undefined для insight). */
function myValue(my: MyReactions | undefined, axis: ReactionAxis): number | undefined {
  if (!my) return undefined;
  if (axis === "agreement") return my.agreement ?? undefined;
  if (axis === "quality") return my.quality ?? undefined;
  return my.insight ? 1 : undefined;
}

function axisCount(summary: ReactionSummary | undefined, axis: ReactionAxis): string {
  if (!summary) return "";
  if (axis === "insight") return summary.insight ? `★ ${summary.insight}` : "★";
  const c = axis === "agreement" ? summary.agreement : summary.quality;
  const pos = c?.positive ?? 0;
  const neg = c?.negative ?? 0;
  return `+${pos} / −${neg}`;
}

export function CommentReactions({
  commentId,
  reactions,
  myReactions,
  allowedAxes,
  canReact,
}: Props) {
  const [pending, startTransition] = useTransition();
  const [error, setError] = useState<string | null>(null);
  // Локальное зеркало моих реакций для оптимистичного UI.
  const [my, setMy] = useState<MyReactions | undefined>(myReactions);

  if (allowedAxes.length === 0) return null;

  function toggle(axis: ReactionAxis, value: number) {
    if (!canReact || pending) return;
    setError(null);
    const current = myValue(my, axis);
    const isSame = current === value;
    // Оптимистично.
    const optimistic: MyReactions = { ...my };
    if (axis === "insight") optimistic.insight = isSame ? false : true;
    else if (axis === "agreement") optimistic.agreement = isSame ? undefined : value;
    else optimistic.quality = isSame ? undefined : value;
    setMy(optimistic);

    startTransition(async () => {
      const result = isSame
        ? await removeReaction({ id: commentId, axis })
        : await setReaction({ id: commentId, axis, value });
      if (!result.success) {
        setMy(myReactions); // откат
        setError(
          result.code === "forbidden"
            ? "У вас нет прав на реакцию."
            : result.error,
        );
      }
    });
  }

  return (
    <div className="flex flex-col gap-1">
      <div className="flex flex-wrap items-center gap-3 text-xs">
        {allowedAxes.map((axis) => {
          const current = myValue(my, axis);
          const values = axis === "insight" ? [1] : [1, -1];
          return (
            <span key={axis} className="flex items-center gap-1">
              <span className="text-(--color-description)">{axisLabel(axis)}:</span>
              {values.map((v) => {
                const active = current === v;
                return (
                  <button
                    key={v}
                    type="button"
                    disabled={!canReact || pending}
                    onClick={() => toggle(axis, v)}
                    aria-pressed={active}
                    className={
                      active
                        ? "rounded border border-(--color-border) bg-(--color-text-pane) px-1.5"
                        : "rounded border border-(--color-border) px-1.5 hover:bg-(--color-text-pane) disabled:opacity-40"
                    }
                  >
                    {axis === "insight" ? "★" : v === 1 ? "+" : "−"}
                  </button>
                );
              })}
              <span className="text-(--color-description)">{axisCount(reactions, axis)}</span>
            </span>
          );
        })}
      </div>
      {error && <p className="text-xs text-red-600">{error}</p>}
    </div>
  );
}

// src/features/comments/ui/comment-tree.tsx
import { CommentNode } from "./comment-node";
import type { Comment, CommentSchema, RootSubtree } from "../types";

/** Строит map parent_id → children из плоского списка узлов. */
function groupByParent(nodes: Comment[]): Map<string | null, Comment[]> {
  const map = new Map<string | null, Comment[]>();
  for (const n of nodes) {
    const key = n.parent_id ?? null;
    const arr = map.get(key) ?? [];
    arr.push(n);
    map.set(key, arr);
  }
  return map;
}

interface BranchProps {
  node: Comment;
  childrenMap: Map<string | null, Comment[]>;
  lectureId: string;
  schema: CommentSchema;
}

async function Branch({ node, childrenMap, lectureId, schema }: BranchProps) {
  const kids = childrenMap.get(node.id) ?? [];
  return (
    <li className="flex flex-col gap-2">
      <CommentNode comment={node} lectureId={lectureId} schema={schema} />
      {kids.length > 0 && (
        <ul className="ml-4 flex flex-col gap-2 border-l border-(--color-border) pl-3">
          {kids.map((kid) => (
            <Branch
              key={kid.id}
              node={kid}
              childrenMap={childrenMap}
              lectureId={lectureId}
              schema={schema}
            />
          ))}
        </ul>
      )}
    </li>
  );
}

interface Props {
  subtrees: RootSubtree[];
  lectureId: string;
  schema: CommentSchema;
}

export function CommentTree({ subtrees, lectureId, schema }: Props) {
  if (subtrees.length === 0) {
    return <p className="text-sm text-(--color-description)">Комментариев пока нет.</p>;
  }
  return (
    <ul className="flex flex-col gap-3">
      {subtrees.flatMap((st) => {
        const root = st.root;
        if (!root) return [];
        const all = [...(st.descendants ?? [])];
        const childrenMap = groupByParent(all);
        return [
          <Branch
            key={root.id}
            node={root}
            childrenMap={childrenMap}
            lectureId={lectureId}
            schema={schema}
          />,
        ];
      })}
    </ul>
  );
}

import { useTransition, type FormEvent } from "react";
import { Button, TextInput } from "@/components/ui";

  /** Текущее значение из searchParams (?cq=). */
  defaultQuery?: string;

/**
 * Поиск по комментариям лекции. Кладёт q в URL (?cq=), страница лекции
 * читает его, дёргает searchComments на сервере и рендерит результаты.
 * Показывается только залогиненным (бек: requiredAuth) — гейт на странице.
 */
export function CommentSearch({ defaultQuery = "" }: Props) {
  const router = useRouter();
  const pathname = usePathname();
  const searchParams = useSearchParams();
  const [pending, startTransition] = useTransition();

  function onSubmit(e: FormEvent<HTMLFormElement>) {
    e.preventDefault();
    const fd = new FormData(e.currentTarget);
    const q = String(fd.get("cq") ?? "").trim();
    const params = new URLSearchParams(searchParams.toString());
    if (q) params.set("cq", q);
    else params.delete("cq");
    const qs = params.toString();
    startTransition(() => router.replace(qs ? `${pathname}?${qs}` : pathname));
  }

  return (
    <form onSubmit={onSubmit} className="flex items-end gap-2">
      <TextInput
        name="cq"
        defaultValue={defaultQuery}
        placeholder="Поиск по комментариям…"
        maxLength={200}
        aria-label="Поиск по комментариям лекции"
      />
      <Button type="submit" disabled={pending}>
        {pending ? "…" : "Найти"}
      </Button>
    </form>
  );
}

  canModerateComments,

  AdminCommentRow,
} from "@/features/comments";

  searchParams: Promise<{ lecture_id?: string }>;
}

/**
 * Admin-модерация комментариев ПЕР-ЛЕКЦИОННАЯ: бек требует lecture_id
 * (GET /api/admin/comments → 422 без него; глобального списка нет).
 * Поэтому страница начинается с выбора лекции (ввод lecture_id) и
 * показывает список только когда он задан.

export default async function AdminCommentsPage({ searchParams }: Props) {

  if (!canModerateComments(me)) forbidden();

  const list = lecture_id
    ? await getAdminLectureComments(lecture_id)

          <span className="text-xs text-(--color-description)">ID лекции</span>
          <TextInput
            name="lecture_id"
            defaultValue={lecture_id ?? ""}
            placeholder="UUID лекции"
          />
        </label>
        <Button type="submit">Показать</Button>
      </form>

      {!lecture_id && (
        <p className="text-sm text-(--color-description)">
          Укажите ID лекции — глобального списка комментариев на бекенде нет.
        </p>
      )}

      {list && (
        <div className="flex flex-col gap-3">
          <p className="text-xs text-(--color-description)">Всего: {list.total}</p>
          {list.items.length === 0 ? (
            <p className="text-sm text-(--color-description)">Комментариев нет.</p>
          ) : (
            list.items.map((c) => <AdminCommentRow key={c.id} comment={c} />)
          )}
        </div>
      )}
    </section>
  );
}

## Риски / допущения

1. **Создание якоря через UI не реализуется** (только чтение/резолв). Спека для comments просит резолв якорей для контекста, а не selection-flow. Создание комментария идёт без `anchor`. Если потребуется привязка к выделению текста — отдельная задача (нужен cross-entity selection capture, выходит за бюджет фичи).
2. **`comment.create` отсутствует в union `Capability`** (`src/utils/permissions.ts` — запретная зона). Используем локальный чек `me.capabilities.includes("comment.create")` с TODO. Добавление в union — отдельный foundation-update PR (как в волне 1). При мерже foundation-update заменить на `can(me, "comment.create")`.
3. **Foundation-touch для comments НЕ требуется** — admin-sidebar уже использует `comment.delete_any` и ссылку `/admin/comments` (проверено `src/app/admin/layout.tsx:48`). Спека §10.6 (расхождение `comment.moderate`) уже устранена в волне 1.
4. **.md/.txt-ссылки указывают прямо на `API_URL`** (server-component читает env). Это работает, т.к. роуты публичные. Если в проде `API_URL` — внутренний адрес, недоступный браузеру, ссылки сломаются — тогда понадобится прокси-роут (как у events). Зафиксировано как допущение; при необходимости добавить `src/app/comments/[id]/export/route.ts` и `src/app/lectures/[id]/comments-export/route.ts` (вне текущего скоупа).
5. **Реакции оптимистичны, но не пересчитывают агрегаты** (`reactions` summary) до revalidation — счётчики «+N/−N» обновятся после `revalidateEntity` на следующем рендере. Это осознанное упрощение (полный optimistic-пересчёт сумм усложнил бы client-стейт).
6. **Поиск кладёт `q` в общий URL лекции (`?cq=`)** и при наличии запроса прячет дерево, показывая результаты. При пустом `cq` — обычное дерево. Это соответствует SSR-паттерну §3.5; для гостя поиск-форма скрыта (бек 401).
7. **`Select`/`ConfirmDialog`/`toast` из UI-kit** — точные сигнатуры свериь при исполнении (задача 11 step 4). Если API отличается — адаптировать без правки UI-kit (запретная зона).
8. **Параллельная работа на странице лекции**: `src/app/lectures/[id]/page.tsx` зарезервирован за comments на всю волну 2 (см. Parallel-safety contract). annotations-агрегатор и documents/media-блоки добавляются ПОЗЖЕ (follow-up annotations / волна 3) — не в этой ветке.
