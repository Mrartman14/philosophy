# План реализации фичи `media`

> **Для агентных исполнителей:** ОБЯЗАТЕЛЬНЫЙ СУБ-СКИЛЛ: используй superpowers:subagent-driven-development (рекомендуется) или superpowers:executing-plans для выполнения этого плана задача-за-задачей. Шаги отмечены чекбоксами (`- [ ]`).

**Goal:** Слайс `src/features/media/` + страницы `/media/my` и `/media/[id]`: загрузка медиа (video/audio через multipart-upload), просмотр (нативный `<video>`/`<audio>`-плеер), удаление (owner / admin `media.delete_any` — независимо от видимости), смена видимости (private→public only), read-only список лекций-контейнеров.

**Architecture:** SSR-first слайс по конвенциям проекта (`docs/frontend-conventions.md`). Server-only `api.ts` (фетчеры на `React.cache`), `actions.ts` (мутации). **Создание медиа — multipart-upload**: бекенд принимает `POST /api/media` с `multipart/form-data` (поля `type` + `file`), НЕ URL-референс (источник истины — `philosophy-api/internal/media/handler.go:CreateMedia`, `service.go:validateAndPersist`). openapi-fetch не умеет аккуратно сериализовать `File` в multipart, поэтому загрузка идёт через `"use server"`-экшен с raw `fetch(body: FormData)` и токеном из cookie — точная копия паттерна `src/components/ast-editor/upload/upload-image.ts`. RBAC: `requireCapability(me, canCreateMedia)` в actions, `canX()` в server-components, boolean-пропсы в client. Локальная пагинация (ui-kit `Pagination` теряет query-параметры).

**Tech Stack:** Next.js 16 (App Router, server components + server actions), React 19, TypeScript (`exactOptionalPropertyTypes`), Zod, openapi-fetch, Vitest, Tailwind v4 (CSS-переменные).

## Контекст бекенда (источник истины)

Прочитано из `/Users/alexander.borisenko/Documents/philosophy-api`:

- **Модель** (`internal/media/model.go`): `Media { id, owner_id, visibility (private|public), type (video|audio), filename, url, created_at }`. `MediaSummary` (для picker `GET /api/media`) — то же без `url`. Поле `visibility` **есть** в модели (комментарии бекенда местами устаревшие — но `SetVisibility` и `PUBLIC_IMMUTABLE` реальны: §6.1 спеки).
- **Создание** (`handler.go:CreateMedia`, `service.go:validateAndPersist`): `POST /api/media`, `multipart/form-data`, поля `type` (`video`|`audio`) и `file`. Медиа всегда создаётся **free-floating** и **private** (visibility выставляется НЕ при создании; новый файл приватен). Валидация расширений (`validate.go`): video → `.mp4/.webm`, audio → `.mp3/.m4a/.ogg`; MIME-сниффинг по первым 512 байтам. Лимит формы — 100 MB. Гейт `media.create` (есть у RoleUser **и** RoleAdmin).
- **Просмотр** (`GetMedia`): `GET /api/media/{media_id}` — optional-auth + shareTokenMW. Возвращает `media.Media` с подписанным `url`. 404 = secure-by-obscurity (не видно ≡ не существует).
- **Удаление** (`Delete`): `DELETE /api/media/{media_id}` — owner ИЛИ `media.delete_any`. Для media `delete_any` действует **независимо от видимости** (§6.2). Не owner и не admin → 403.
- **Видимость** (`SetVisibility`): `PATCH /api/media/{media_id}/visibility`, body `{ visibility }`. Только владелец. `public→private` → 422 `PUBLIC_IMMUTABLE`. Не-владельцу приватного → 404, публичного → 403.
- **Мои медиа** (`ListMyMedia`): `GET /api/me/media?offset&limit&free_floating` — default limit 20, max 100. Возвращает `media.Media[]` + pagination.
- **Picker** (`Search`): `GET /api/media?q&type&offset&limit` — `MediaSummary[]`. Неизвестный `type` → 422 `INVALID_FILE_TYPE`. **В этом плане picker НЕ строим** (его потребитель — attachments-пикеры из веток documents/lecture-enrichment, волна 3). Типизацию picker-фетчера не делаем.
- **Attachments** (`GET /api/media/{id}/attachments`): возвращает `attachment.AttachmentDTO[]` (`container_type`, `container_id`, `entity_type`, `entity_id`, `sort_order`, `attached_at`) — **только ID контейнеров, без заголовков лекций**. Read-only. См. §10.3 — admin-списка media НЕТ, sidebar-пункта не будет.
- **RBAC** (`internal/rbac/capabilities.go`): `media.create` (RoleUser+RoleAdmin), `media.delete_any` (только RoleAdmin). Оба подтверждены.

### Критические уроки волны 1 (соблюдать дословно)

1. **Uppercase-коды ошибок**: бек шлёт `code` в `UPPER_SNAKE_CASE` (`FORBIDDEN`, `PUBLIC_IMMUTABLE`, `SUSPENDED`, `VALIDATION_ERROR`, `INVALID_FILE_TYPE`). Маппить через `rethrowApiError` (образец — `events/actions.ts`, `preferences/actions.ts`).
2. **`exactOptionalPropertyTypes`**: опциональные поля типизируем `?: T | undefined`; в body API подставляем поле только если оно определено (паттерн `...(x ? { x } : {})`).
3. **Branded 403-тексты**: в UI на `result.code === "forbidden"` показываем «У вас нет прав на <действие>.», не raw `result.error`.
4. **`import "server-only";`** первой строкой в `api.ts`, `actions.ts`, `permissions.ts`, `schemas.ts`.
5. **`createFormAction`/`parseFormData`/`requireCapability`/`revalidateEntity`** — обязательны. Для не-FormData входа (удаление по id, смена видимости, multipart-upload) — `createAction` (см. `preferences/actions.ts`). Multipart-upload — отдельный паттерн (raw fetch, см. ниже).
6. **Локальная пагинация**: ui-kit `Pagination` строит href только из offset и теряет фильтры — копируем слайс-локальный `MediaPagination` (образец `audit/ui/audit-pagination.tsx`).

## Решения и допущения (читать до старта)

1. **Создание = multipart-upload, не URL.** Подтверждено бекендом. Используем raw-fetch-паттерн `upload-image.ts`, потому что openapi-fetch не сериализует `File` в multipart надёжно. Это отдельный `"use server"`-файл `upload-media.ts` (не идёт через `createFormAction`, т.к. нам нужен прямой контроль над телом и статус-кодами 413/422).
2. **Новое медиа всегда private.** Бекенд создаёт free-floating private. Поэтому форма загрузки НЕ предлагает выбор видимости. Видимость меняется отдельно на detail-странице (private→public only).
3. **`media-containers` (список лекций-контейнеров) — локальный read-only компонент слайса.** Generic `src/components/attachments/` владеет ветка `documents` и может быть ещё не смержена к моменту мержа media. Поэтому делаем простой локальный список из `GET /api/media/{id}/attachments`. Бекенд отдаёт только `container_id` (без заголовка лекции) — рендерим как ссылку `/lectures/{container_id}` с подписью «Лекция {короткий-id}». Это осознанное упрощение; обогащение заголовками — задача lecture-enrichment (волна 3). **Не импортировать `@/components/attachments`** — ESLint и отсутствие модуля сломают сборку.
4. **Picker (`GET /api/media`) НЕ реализуем** — его потребитель (attachments-пикеры) появляется в волне 3. Типизацию и фетчер picker не добавляем (YAGNI).
5. **`GET /api/lectures/{id}/media` НЕ реализуем** — это волна 3 (lecture-enrichment), не сейчас (§ промпта).
6. **`/media/[id]` — target mark `media_ref`** (ast-render уже резолвит `media_ref` в ссылку на `/media/{id}` — ветка ast-editor-phase-2). Наша задача — чтобы страница существовала и корректно рендерила плеер.
7. **Кеш.** `getMyMedia` — per-user, постоянно меняется → только `React.cache`, без `unstable_cache` (как `audit/api.ts`). `getMediaById` — оборачиваем в `unstable_cache` с тегом `media:{id}` для инвалидации после смены видимости; ключ включает id. Тег `MEDIA` добавляем в `src/api/tags.ts`. После мутаций — `revalidateEntity(Tags.MEDIA, id)` и `revalidateEntity(Tags.MEDIA)`.

- [ ] **Step 3: Написать `permissions.ts`**

`canCreateMedia` / `canDeleteAnyMedia` — плоские capability-чеки через `can()` (status-гейт внутри). `canDeleteMedia` — owner ИЛИ delete_any (независимо от видимости, §6.2). `canChangeMediaVisibility` — только owner И только если текущая видимость `private` (public→private запрещён, §6.1). Owner-чек требует active-статус: оборачиваем в `isMutationAllowed`.

```ts
// src/features/media/permissions.ts
import "server-only";
import type { MaybeMe } from "@/utils/me";
import { can, isMutationAllowed } from "@/utils/permissions";
import type { Media } from "./types";

/**
 * Имена capabilities — строго из philosophy-api internal/rbac/capabilities.go:
 *   media.create     — RoleUser + RoleAdmin
 *   media.delete_any — только RoleAdmin
 */

/** POST /api/media — загрузка нового медиа. */
export function canCreateMedia(me: MaybeMe): boolean {
  return can(me, "media.create");
}

/** Бланкетное admin-удаление любого медиа (для UI-веток admin-кнопок). */
export function canDeleteAnyMedia(me: MaybeMe): boolean {
  return can(me, "media.delete_any");
}

/**
 * DELETE /api/media/{id}. Бек: owner ИЛИ media.delete_any. Для media
 * delete_any действует НЕЗАВИСИМО от видимости (§6.2 спеки;
 * philosophy-api internal/media/service.go:Delete).
 */
export function canDeleteMedia(me: MaybeMe, media: Media): boolean {
  if (!isMutationAllowed(me)) return false;
  if (media.owner_id === me.id) return true;
  return can(me, "media.delete_any");
}

/**
 * PATCH /api/media/{id}/visibility. Бек: только владелец, и только
 * private→public (public иммутабелен вниз, 422 PUBLIC_IMMUTABLE; §6.1).
 * Admin НЕ редактирует чужой контент (admin power split, §6.2) — поэтому
 * без delete_any-override здесь.
 */
export function canChangeMediaVisibility(me: MaybeMe, media: Media): boolean {
  if (!isMutationAllowed(me)) return false;
  if (media.owner_id !== me.id) return false;
  return media.visibility === "private";
}
```

```ts
// src/features/media/api.ts
import "server-only";
import { cache } from "react";
import { unstable_cache } from "next/cache";
import { createApiClient } from "@/api/client";
import { Tags } from "@/api/tags";
import type { Media, MediaAttachment } from "./types";

export interface MyMediaFilter {
  offset?: number;
  /** Бек: default 20, max 100. */
  limit?: number;
  /** Только незакреплённые (без вхождения в attachments). */
  freeFloating?: boolean;
}

export interface MyMediaResult {
  items: Media[];
  total: number;
  offset: number;
  limit: number;
}

/**
 * GET /api/me/media — мои медиа. Per-user, постоянно меняется → только
 * React.cache (дедуп в рамках запроса), без unstable_cache (как audit/api.ts).
 */
export const getMyMedia = cache(
  async (filter: MyMediaFilter = {}): Promise<MyMediaResult> => {
    const api = await createApiClient();
    const offset = filter.offset ?? 0;
    const limit = filter.limit ?? 20;
    const query: {
      offset: number;
      limit: number;
      free_floating?: boolean;
    } = { offset, limit };
    if (filter.freeFloating) query.free_floating = true;
    const { data, error } = await api.GET("/api/me/media", {
      params: { query },
    });
    if (error) {
      throw new Error(error.error ?? "Не удалось загрузить медиа");
    }
    return {
      items: (data?.data ?? []) as Media[],
      total: data?.pagination?.total ?? 0,
      offset: data?.pagination?.offset ?? offset,
      limit: data?.pagination?.limit ?? limit,
    };
  },
);

/**
 * GET /api/media/{media_id} — одно медиа с подписанным url. 404 →
 * возвращаем null (secure-by-obscurity: «не видно» ≡ «не существует»).
 * Кешируется per-id; инвалидация — revalidateEntity(Tags.MEDIA, id) после
 * смены видимости. unstable_cache требует, чтобы id попал и в keyParts, и в
 * tags — иначе все id делят один кеш.
 */
export const getMediaById = cache(
  async (id: string): Promise<Media | null> => {
    const fetcher = unstable_cache(
      async (mediaId: string): Promise<Media | null> => {
        const api = await createApiClient();
        const { data, error, response } = await api.GET(
          "/api/media/{media_id}",
          { params: { path: { media_id: mediaId } } },
        );
        if (response.status === 404) return null;
        if (error) {
          throw new Error(error.error ?? "Не удалось загрузить медиа");
        }
        return (data?.data ?? null) as Media | null;
      },
      ["media-by-id", id],
      { tags: [`${Tags.MEDIA}:${id}`] },
    );
    return fetcher(id);
  },
);

/**
 * GET /api/media/{id}/attachments — контейнеры (лекции), к которым привязано
 * медиа. Read-only. Бек отдаёт только container_id (без заголовков). 404 →
 * пустой список (медиа без attachments).
 */
export const getMediaContainers = cache(
  async (id: string): Promise<MediaAttachment[]> => {
    const api = await createApiClient();
    const { data, error, response } = await api.GET(
      "/api/media/{id}/attachments",
      { params: { path: { id } } },
    );
    if (response.status === 404) return [];
    if (error) {
      throw new Error(error.error ?? "Не удалось загрузить контейнеры");
    }
    return (data?.data ?? []) as MediaAttachment[];
  },
);
```

```ts
// src/features/media/actions.ts
"use server";
import "server-only";
import { createApiClient } from "@/api/client";
import { createAction } from "@/utils/create-action";
import { getMe } from "@/utils/me";
import { ForbiddenError, requireCapability } from "@/utils/permissions";
import { revalidateEntity } from "@/utils/revalidate";
import { Tags } from "@/api/tags";
import { canDeleteMedia, canChangeMediaVisibility } from "./permissions";
import { MediaIdSchema, MediaVisibilitySchema } from "./schemas";
import type { Media } from "./types";

type ApiError = { code?: string; error?: string };

/** Маппинг UPPER_SNAKE_CASE-кодов бекенда на доменные ошибки фронта. */
function rethrowApiError(err: ApiError | undefined): never {
  switch (err?.code) {
    case "FORBIDDEN":
      throw new ForbiddenError("role", err.error);
    case "SUSPENDED":
      throw new ForbiddenError("status", err.error);
    case "PUBLIC_IMMUTABLE":
      throw new Error(
        "Публичное медиа нельзя сделать приватным. Удалите и загрузите заново.",
      );
    case "NOT_FOUND":
      throw new Error("Медиа не найдено.");
  }
  throw new Error(err?.error ?? "Ошибка сервера");
}

/** Загружает media-запись для owner-aware RBAC. 404 → ForbiddenError (secure). */
async function loadMediaForGate(id: string): Promise<Media> {
  const api = await createApiClient();
  const { data, error, response } = await api.GET("/api/media/{media_id}", {
    params: { path: { media_id: id } },
  });
  if (response.status === 404) {
    // Не видно ≡ не существует. Для гейта это отказ.
    throw new ForbiddenError("owner", "Медиа не найдено");
  }
  if (error) rethrowApiError(error);
  return (data?.data ?? null) as Media;
}

/**
 * DELETE /api/media/{id}. Owner ИЛИ media.delete_any (независимо от
 * видимости, §6.2). Используется и владельцем со страницы /media/[id], и
 * админом (admin-списка нет — §10.3).
 */
export const deleteMedia = createAction(async (rawId: string) => {
  const me = await getMe();
  const { id } = MediaIdSchema.parse({ id: rawId });
  const media = await loadMediaForGate(id);
  requireCapability(me, (m) => canDeleteMedia(m, media));
  const api = await createApiClient();
  const { error } = await api.DELETE("/api/media/{media_id}", {
    params: { path: { media_id: id } },
  });
  if (error) rethrowApiError(error as ApiError);
  revalidateEntity(Tags.MEDIA, id);
  revalidateEntity(Tags.MEDIA);
  return undefined;
});

/**
 * PATCH /api/media/{id}/visibility. Только владелец, только private→public.
 * Вход — { id, visibility } (visibility всегда "public" из UI).
 */
export const setMediaVisibility = createAction(
  async (raw: { id: string; visibility: string }) => {
    const me = await getMe();
    const input = MediaVisibilitySchema.parse(raw);
    const media = await loadMediaForGate(input.id);
    requireCapability(me, (m) => canChangeMediaVisibility(m, media));
    const api = await createApiClient();
    const { error } = await api.PATCH("/api/media/{media_id}/visibility", {
      params: { path: { media_id: input.id } },
      body: { visibility: input.visibility as "private" | "public" },
    });
    if (error) rethrowApiError(error as ApiError);
    revalidateEntity(Tags.MEDIA, input.id);
    revalidateEntity(Tags.MEDIA);
    return undefined;
  },
);
```

```ts
// src/features/media/upload-media.ts
"use server";
import "server-only";
import { cookies } from "next/headers";
import { revalidateTag } from "next/cache";
import { getMe } from "@/utils/me";
import { Tags } from "@/api/tags";
import { canCreateMedia } from "./permissions";
import type { Media } from "./types";

const API_URL = process.env.API_URL ?? "http://localhost:8080";

export type UploadMediaResult =
  | { success: true; data: Media }
  | {
      success: false;
      error: string;
      code: "forbidden" | "file_too_large" | "invalid_file";
    }
  | { success: false; error: string; code?: undefined };

interface ApiError {
  code?: string;
  error?: string;
}

/**
 * POST /api/media — multipart-upload (поля type + file). Бек создаёт медиа
 * free-floating и private. Идёт через raw fetch (а не openapi-fetch), потому
 * что openapi-fetch не сериализует File в multipart надёжно — тот же паттерн,
 * что src/components/ast-editor/upload/upload-image.ts.
 *
 * Ожидает FormData с полями:
 *   - file: File (видео .mp4/.webm или аудио .mp3/.m4a/.ogg)
 *   - type: "video" | "audio"
 */
export async function uploadMedia(
  formData: FormData,
): Promise<UploadMediaResult> {
  // Ранний RBAC-отказ (defense-in-depth; бек тоже гейтит media.create).
  const me = await getMe();
  if (!canCreateMedia(me)) {
    return {
      success: false,
      error: "У вас нет прав на загрузку медиа.",
      code: "forbidden",
    };
  }

  const file = formData.get("file");
  const type = formData.get("type");
  if (!(file instanceof File) || file.size === 0) {
    return { success: false, error: "Выберите файл." };
  }
  if (type !== "video" && type !== "audio") {
    return { success: false, error: "Выберите тип: видео или аудио." };
  }

  const cookieStore = await cookies();
  const token = cookieStore.get("token")?.value;

  let res: Response;
  try {
    res = await fetch(`${API_URL}/api/media`, {
      method: "POST",
      headers: token ? { Authorization: `Bearer ${token}` } : {},
      body: formData,
    });
  } catch (e) {
    return {
      success: false,
      error: e instanceof Error ? e.message : "Сетевая ошибка",
    };
  }

  if (res.status === 201) {
    const body = (await res.json()) as { data?: Media } | Media;
    const media =
      typeof body === "object" && body !== null && "data" in body
        ? (body as { data: Media }).data
        : (body as Media);
    revalidateTag(Tags.MEDIA, "default");
    return { success: true, data: media };
  }

  let body: ApiError = {};
  try {
    body = (await res.json()) as ApiError;
  } catch {
    /* пустое/не-JSON тело — фоллбек на статус */
  }

  if (res.status === 401 || res.status === 403) {
    return {
      success: false,
      error: body.error ?? "Нет доступа.",
      code: "forbidden",
    };
  }
  if (res.status === 413) {
    return {
      success: false,
      error: body.error ?? "Файл слишком большой (макс 100 MB).",
      code: "file_too_large",
    };
  }
  // 400/422 — невалидное расширение/MIME/тип (validate.go).
  if (res.status === 400 || res.status === 422) {
    return {
      success: false,
      error:
        body.error ??
        "Неподдерживаемый формат. Видео: mp4/webm. Аудио: mp3/m4a/ogg.",
      code: "invalid_file",
    };
  }
  return {
    success: false,
    error: body.error ?? `Ошибка загрузки: ${res.status}`,
  };
}
```

```tsx
"use client";
// src/features/media/ui/media-pagination.tsx
import Link from "next/link";
import { usePathname, useSearchParams } from "next/navigation";

interface Props {
  offset: number;
  limit: number;
  total: number;
}

const linkCls =
  "rounded border border-(--color-border) px-3 py-1 hover:bg-(--color-text-pane)";
const disabledCls =
  "rounded border border-(--color-border) px-3 py-1 opacity-40";

/**
 * Слайс-локальная пагинация. ui-kit Pagination строит href только из offset
 * и теряет прочие searchParams; src/components/ui — запретная зона. Этот
 * компонент мержит offset в ТЕКУЩИЙ query (например, free_floating выживает).
 */
export function MediaPagination({ offset, limit, total }: Props) {
  const pathname = usePathname();
  const searchParams = useSearchParams();

  function makeHref(nextOffset: number): string {
    const params = new URLSearchParams(searchParams.toString());
    if (nextOffset > 0) params.set("offset", String(nextOffset));
    else params.delete("offset");
    const qs = params.toString();
    return qs ? `${pathname}?${qs}` : pathname;
  }

  const hasPrev = offset > 0;
  const hasNext = offset + limit < total;

  return (
    <nav aria-label="Пагинация" className="flex items-center gap-2 text-sm">
      {hasPrev ? (
        <Link href={makeHref(Math.max(0, offset - limit))} className={linkCls}>
          ← Назад
        </Link>
      ) : (
        <span className={disabledCls}>← Назад</span>
      )}
      <span className="text-(--color-description)">
        {total === 0
          ? "0 из 0"
          : `${offset + 1}–${Math.min(offset + limit, total)} из ${total}`}
      </span>
      {hasNext ? (
        <Link href={makeHref(offset + limit)} className={linkCls}>
          Вперёд →
        </Link>
      ) : (
        <span className={disabledCls}>Вперёд →</span>
      )}
    </nav>
  );
}
```

```tsx
"use client";
// src/features/media/ui/media-upload-form.tsx
import { useRef, useState, useTransition } from "react";
import { useRouter } from "next/navigation";
import { Button, useToast } from "@/components/ui";
import { uploadMedia } from "../upload-media";

interface MediaUploadFormProps {
  /** canCreateMedia(me) со страницы. Если false — форма не рендерится. */
  canUpload: boolean;
}

const ACCEPT = ".mp4,.webm,.mp3,.m4a,.ogg,video/*,audio/*";

/**
 * Форма загрузки медиа. type выбирается явно (бек требует точный video|audio).
 * Новое медиа создаётся приватным (бек: free-floating + private), поэтому
 * выбора видимости здесь нет — публикация делается на /media/[id].
 */
export function MediaUploadForm({ canUpload }: MediaUploadFormProps) {
  const router = useRouter();
  const toast = useToast();
  const inputRef = useRef<HTMLInputElement>(null);
  const [type, setType] = useState<"video" | "audio">("video");
  const [pending, startTransition] = useTransition();
  const [busy, setBusy] = useState(false);

  if (!canUpload) return null;

  async function handleSubmit(e: React.FormEvent<HTMLFormElement>) {
    e.preventDefault();
    const file = inputRef.current?.files?.[0];
    if (!file) {
      toast.add({ title: "Выберите файл" });
      return;
    }
    const fd = new FormData();
    fd.set("file", file);
    fd.set("type", type);
    setBusy(true);
    const result = await uploadMedia(fd);
    setBusy(false);
    if (!result.success) {
      const description =
        result.code === "forbidden"
          ? "У вас нет прав на загрузку медиа."
          : result.error;
      toast.add({ title: "Ошибка загрузки", description });
      return;
    }
    toast.add({ title: "Загружено", description: result.data.filename });
    if (inputRef.current) inputRef.current.value = "";
    startTransition(() => router.refresh());
  }

  return (
    <form onSubmit={handleSubmit} className="flex flex-col gap-3">
      <div className="flex flex-col gap-1">
        <label htmlFor="media-type" className="text-sm font-medium">
          Тип
        </label>
        <select
          id="media-type"
          value={type}
          onChange={(e) => setType(e.target.value as "video" | "audio")}
          className="rounded border border-(--color-border) bg-transparent px-3 py-2"
        >
          <option value="video">Видео (mp4, webm)</option>
          <option value="audio">Аудио (mp3, m4a, ogg)</option>
        </select>
      </div>
      <div className="flex flex-col gap-1">
        <label htmlFor="media-file" className="text-sm font-medium">
          Файл
        </label>
        <input
          id="media-file"
          ref={inputRef}
          type="file"
          accept={ACCEPT}
          className="text-sm"
        />
      </div>
      <Button type="submit" className="self-start" disabled={busy || pending}>
        {busy ? "Загрузка…" : "Загрузить"}
      </Button>
      <p className="text-xs text-(--color-description)">
        Новое медиа создаётся приватным. Опубликовать можно на его странице.
      </p>
    </form>
  );
}
```

## Риски и допущения

1. **[Главное] Создание media = multipart-upload, НЕ URL-референс.** Подтверждено кодом бекенда (`internal/media/handler.go:CreateMedia` парсит `multipart/form-data` с полями `type` + `file`; `service.go:validateAndPersist` валидирует расширение/MIME и заливает байты в storage). Никакого «URL медиа» на входе нет. Поэтому загрузка идёт raw-fetch'ем (`upload-media.ts`), как `upload-image.ts`. Если бекенд изменит контракт — переделать только `upload-media.ts` и `media-upload-form.tsx`.
2. **Новое медиа всегда private + free-floating.** Бек не принимает visibility при создании. Форма загрузки не предлагает выбор видимости; публикация — отдельный шаг на `/media/[id]`. Допущение зафиксировано в UI-тексте формы.
3. **Attachments-список отдаёт только `container_id` без заголовков.** `GET /api/media/{id}/attachments` → `AttachmentDTO[]` без title лекции. Рендерим «Лекция <короткий-id>». Обогащение заголовками — задача lecture-enrichment (волна 3). Это осознанное упрощение, не баг.
4. **Локальный `media-containers` вместо generic `@/components/attachments`.** Ветка documents — владелец generic-attachments и может быть не смержена к моменту мержа media (порядок мержей: documents→annotations, media отдельно). Импорт несуществующего/чужого модуля сломал бы сборку и нарушил бы изоляцию слайсов. Решение — простой локальный read-only список. **Если на момент мержа media окажется, что documents уже смержена и generic-компонент стабилен — это не повод переписывать: read-only локальный список самодостаточен; миграцию на generic можно сделать отдельным follow-up без блокировки.**
5. **RBAC для visibility — без admin-override.** Бек: только владелец меняет видимость (`service.go:SetVisibility`, admin power split §6.2). `canChangeMediaVisibility` намеренно не пускает admin к чужому медиа. Это корректно по спеке.
6. **`getMediaById` дважды в `/media/[id]` (generateMetadata + page).** `React.cache` дедуплицирует в рамках одного запроса — реального двойного сетевого вызова нет.
7. **Picker (`GET /api/media`) и `GET /api/lectures/{id}/media` не реализованы** — YAGNI, их потребители появятся в волне 3. Типы `MediaSummary` экспортированы для симметрии, но фетчер picker не пишем.
8. **Foundation-touch (header «Мои медиа») — вне ветки.** Если foundation-агент волны 2 не добавит ссылку, `/media/my` достижима только по прямому URL. Зафиксировано в контракте как явная зависимость.
9. **`revalidateTag` в `upload-media.ts`** вызывается напрямую с `"default"`-профилем (Next 16), а не через `revalidateEntity` — потому что файл и так raw-fetch-уровня; поведение идентично. Альтернатива (импортировать `revalidateEntity`) тоже валидна — исполнитель может выбрать её для единообразия, поведение не меняется.
