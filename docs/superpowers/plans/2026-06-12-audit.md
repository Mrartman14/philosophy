> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Страница `/admin/audit` — просмотр журнала админ-действий (`GET /api/admin/audit`) с фильтрами через URL searchParams (actor, target_type, target_id, action, from, to), offset/limit-пагинацией и компактным разворотом `details`-JSON.

Страница `src/app/admin/audit/page.tsx` — Layer-3 гейт `getMe() + canReadAudit + forbidden()`, парсит searchParams схемой и компонует слайс.

**Tech Stack:** Next.js App Router (server components + searchParams), openapi-fetch (`createApiClient`), Zod v4, UI-kit `@/components/ui` (Table, Select, TextInput, Button, EmptyState), Vitest.

## Справка по бекенду (сверено с philosophy-api, не только schema.ts)

- **Эндпоинт:** `GET /api/admin/audit` (`internal/audit/handler.go`). Auth обязателен; внутри хендлера — `rbac.HasCapability(actor.Role, rbac.CapAuditRead)`; отказ → 403, без токена → 401.
- **Capability:** `audit.read` (`internal/rbac/capabilities.go:35`, `CapAuditRead`). Присутствует в `components["schemas"]["rbac.Capability"]` schema.ts, но ОТСУТСТВУЕТ в узком union `Capability` фронта (`src/utils/permissions.ts`, запретная зона) — см. «Осознанные отклонения», п. 2.
- **Query-параметры (точные имена из handler.go):** `actor` (точное равенство `actor_user_id`, UUID; по username бек НЕ ищет), `target_type` (точное равенство), `target_id` (точное равенство), `action` (точное равенство), `from`/`to` (RFC3339; невалидные значения бек молча игнорирует), `offset` (default 0), `limit` (default 50, max 100 — `httputil.ParsePagination(r, 50, 100)`).
- **Ответ:** `httputil.ListResponse & { data?: audit.Record[] }` + `pagination { offset, limit, total }`; `total` — без учёта offset/limit; сортировка `created_at DESC`.
- **`audit.Record`** (в schema.ts все поля optional): `id`, `actor_user_id`, `actor_username` (LEFT JOIN users; у удалённого актора — пустая строка), `action`, `target_type`, `target_id`, `details` (объект), `request_id`, `created_at` (RFC3339).
- **Формат `action`:** `domain.verb` в snake_case, изредка три сегмента. Реальные примеры из call-sites: `lecture.create`, `lecture.visibility_change`, `lecture.cover.set`, `comment.admin_delete`, `user.role_change`, `user.status_change`, `push.broadcast`, `share_link.create`, `glossary.create`.
- **`target_type`** (полный перечень литералов из call-sites `audit.Logger` в `internal/`): `annotation`, `banner`, `canvas`, `comment`, `document`, `event`, `form`, `glossary_term`, `lecture`, `media`, `push`, `tag`, `trail`, `user`. Динамический случай `share_link.create` пишет `TargetType = ResourceType` ссылки (`lecture|document|trail|media|form|canvas`) — все уже в перечне.
- **`details`** — ТОЛЬКО метаданные действия (идентификаторы block_id/file_id, переходы from→to, filename, type, block_count); пользовательский контент туда не пишется (`internal/audit/audit.go`). Рендерим как pretty-printed JSON в `<details>`-развороте.

## Осознанные отклонения от шаблона `_template` (зафиксировано, не баг)

1. **Нет `actions.ts`.** Фича read-only: бек не даёт мутаций audit-лога. `createFormAction`/`parseFormData`/`requireCapability`/`revalidateEntity` в мутационном смысле не используются — пункт чеклиста шаблона «использует createFormAction…» неприменим.
2. **`canReadAudit` не использует `can()` из `@/utils/permissions`.** Литерал `"audit.read"` отсутствует в union `Capability`, а `src/utils/permissions.ts` — запретная зона (правится только foundation-touch'ем). Хелпер локально повторяет семантику `can()` (гость → false, не-active → false, иначе членство в `capabilities`); константа типизирована `satisfies components["schemas"]["rbac.Capability"]` — typo поймает tsc. Запрет CLAUDE.md «не дублировать status-чек» относится к хелперам ПОВЕРХ `can()`; здесь `can()` недоступен, поэтому status-чек обязан быть локальным. Миграция на `can()` — в foundation-touch.
3. **Слайс-локальная пагинация `AuditPagination`** вместо ui-kit `Pagination`: ui-kit-вариант строит href только из offset и теряет фильтры; его собственный docstring предлагает решать сохранение фильтров на стороне потребителя. `src/components/ui/*` — запретная зона.
4. **Нет тега в `src/api/tags.ts` и `unstable_cache`.** Мутаций нет → `revalidateEntity` нечем дёргать; данные авторизованы per-user (cookie) и постоянно растут — cross-request кеш вреден. Только `React.cache` (дедупликация в рамках запроса).
5. **`target-types.ts` — дополнительный файл слайса** (вне списка шаблона): перечень `target_type` нужен и server-схеме (`schemas.ts`, `import "server-only"`), и client-форме (`ui/audit-filter-form.tsx`). Импорт server-only файла в client-компонент уронил бы сборку, поэтому константа вынесена в нейтральный модуль.

```ts
// src/features/audit/permissions.ts
import "server-only";
import type { components } from "@/api/schema";
import type { MaybeMe } from "@/utils/me";

/**
 * "audit.read" пока отсутствует в узком union Capability
 * (src/utils/permissions.ts — запретная зона, правится только
 * foundation-touch'ем волны), поэтому can() здесь недоступен. Чек локально
 * повторяет его семантику: гость → false, не-active → false, иначе
 * членство в capabilities. Константа типизирована против rbac.Capability
 * из сгенерированного schema.ts — typo ловит tsc.
 *
 * Foundation-touch волны 1: добавить "audit.read" в union Capability и
 * заменить тело на `can(me, "audit.read")`.
 */
const AUDIT_READ = "audit.read" satisfies components["schemas"]["rbac.Capability"];

export function canReadAudit(me: MaybeMe): boolean {
  if (!me) return false;
  if (me.status !== "active") return false;
  return me.capabilities.includes(AUDIT_READ);
}
```

```ts
// src/features/audit/schemas.ts
import "server-only";
import { z } from "zod";
import { AUDIT_TARGET_TYPES } from "./target-types";

/**
 * Валидация фильтров audit-лога из URL searchParams.
 *
 * Строгие суб-схемы экспортируются для прямого unit-тестирования
 * failure-кейсов. В композитной AuditLogFilterSchema каждое поле обёрнуто
 * в .optional().catch(undefined): битый параметр (рукописный URL, устаревшая
 * закладка) молча отбрасывается, страница не падает.
 */

export const AuditActorSchema = z.string().uuid("Некорректный UUID актора");

export const AuditTargetTypeSchema = z.enum(AUDIT_TARGET_TYPES);

/** Формат action на беке: domain.verb (изредка domain.noun.verb), snake_case. */
export const AuditActionSchema = z
  .string()
  .trim()
  .regex(/^[a-z_]+(\.[a-z_]+){1,2}$/, "Формат: domain.verb");

/**
 * Дата из <input type="datetime-local"> ("2026-06-12T13:45") или полный ISO.
 * Преобразуется в RFC3339 UTC, как ждёт GET /api/admin/audit. Строка без
 * таймзоны интерпретируется в TZ процесса Next-сервера — осознанный
 * компромисс для админ-инструмента (см. план, «Риски/допущения»).
 */
export const AuditDateSchema = z.string().transform((s, ctx) => {
  const d = new Date(s);
  if (Number.isNaN(d.getTime())) {
    ctx.addIssue({ code: z.ZodIssueCode.custom, message: "Некорректная дата" });
    return z.NEVER;
  }
  return d.toISOString();
});

export const AuditOffsetSchema = z.coerce.number().int().min(0);

export const AuditLogFilterSchema = z.object({
  actor: AuditActorSchema.optional().catch(undefined),
  target_type: AuditTargetTypeSchema.optional().catch(undefined),
  target_id: z.string().trim().min(1).max(200).optional().catch(undefined),
  action: AuditActionSchema.optional().catch(undefined),
  from: AuditDateSchema.optional().catch(undefined),
  to: AuditDateSchema.optional().catch(undefined),
```

```ts
import "server-only";
import { cache } from "react";
import { createApiClient } from "@/api/client";
import type { AuditRecord } from "./types";

export interface AuditLogFilter {
  /** Точное равенство actor_user_id (UUID). По username бек не ищет. */
  actor?: string;
  target_type?: string;
  target_id?: string;
  /** Точное равенство, формат domain.verb (например, lecture.create). */
  action?: string;
  /** RFC3339. */
  from?: string;
  /** RFC3339. */
  to?: string;
  offset?: number;
  /** Бек: default 50, max 100. */
  limit?: number;
}

export interface AuditLogResult {
  items: AuditRecord[];
  total: number;
  offset: number;
  limit: number;
}

/**
 * Журнал админ-действий. Read-only фича: мутаций нет, revalidateEntity не
 * нужен, поэтому тег в src/api/tags.ts не заводится; unstable_cache не
 * используется (данные авторизованы per-user и постоянно растут).
 * React.cache дедуплицирует вызовы в рамках одного запроса.
 */
export const getAuditLog = cache(
  async (filter: AuditLogFilter = {}): Promise<AuditLogResult> => {
    const api = await createApiClient();
    const offset = filter.offset ?? 0;
    const limit = filter.limit ?? 50;
    const query: {
      offset: number;
      limit: number;
      actor?: string;
      target_type?: string;
      target_id?: string;
      action?: string;
      from?: string;
      to?: string;
    } = { offset, limit };
    if (filter.actor) query.actor = filter.actor;
    if (filter.target_type) query.target_type = filter.target_type;
    if (filter.target_id) query.target_id = filter.target_id;
    if (filter.action) query.action = filter.action;
    if (filter.from) query.from = filter.from;
    if (filter.to) query.to = filter.to;

    const { data, error } = await api.GET("/api/admin/audit", {
      params: { query },
    });
    if (error) {
      throw new Error(error.error ?? "Не удалось загрузить audit-лог");
    }
    return {
      items: (data?.data ?? []) as AuditRecord[],
      total: data?.pagination?.total ?? 0,
      offset: data?.pagination?.offset ?? offset,
      limit: data?.pagination?.limit ?? limit,
    };
  },
);
```

```tsx
  "target_id",
```

```tsx
"use client";
// src/features/audit/ui/audit-pagination.tsx
import Link from "next/link";
import { usePathname, useSearchParams } from "next/navigation";

interface Props {
  offset: number;
  limit: number;
  total: number;
}

const linkCls =
  "rounded border border-(--color-border) px-3 py-1 hover:bg-(--color-text-pane)";
const disabledCls =
  "rounded border border-(--color-border) px-3 py-1 opacity-40";

/**
 * Слайс-локальная пагинация. ui-kit `Pagination` строит href только из
 * offset и теряет фильтры; его docstring явно отдаёт сохранение прочих
 * searchParams на сторону потребителя, а src/components/ui — запретная
 * зона. Этот компонент мержит offset в ТЕКУЩИЙ query — фильтры выживают.
 */
export function AuditPagination({ offset, limit, total }: Props) {
  const pathname = usePathname();
  const searchParams = useSearchParams();

  function makeHref(nextOffset: number): string {
    const params = new URLSearchParams(searchParams.toString());
    if (nextOffset > 0) params.set("offset", String(nextOffset));
    else params.delete("offset");
    const qs = params.toString();
    return qs ? `${pathname}?${qs}` : pathname;
  }

  const hasPrev = offset > 0;
  const hasNext = offset + limit < total;

  return (
    <nav aria-label="Пагинация" className="flex items-center gap-2 text-sm">
      {hasPrev ? (
        <Link href={makeHref(Math.max(0, offset - limit))} className={linkCls}>
          ← Назад
        </Link>
      ) : (
        <span className={disabledCls}>← Назад</span>
      )}
      <span className="text-(--color-description)">
        {total === 0
          ? "0 из 0"
          : `${offset + 1}–${Math.min(offset + limit, total)} из ${total}`}
      </span>
      {hasNext ? (
        <Link href={makeHref(offset + limit)} className={linkCls}>
          Вперёд →
        </Link>
      ) : (
        <span className={disabledCls}>Вперёд →</span>
      )}
    </nav>
  );
}
```

## Риски / допущения

- **TZ дат-фильтров:** значение `<input type="datetime-local">` не несёт таймзоны; `AuditDateSchema` интерпретирует его в TZ процесса Next-сервера. Для админ-инструмента (диапазонная фильтрация журнала) это приемлемо; точную выборку всегда можно сделать, вписав полный ISO с офсетом прямо в URL. Альтернатива (конверсия в браузере) даёт hydration-mismatch и здесь не оправдана.
- **`actor` ищется только по UUID** (`actor_user_id`), не по username — ограничение бекенда; placeholder формы говорит об этом явно.
- **Перечень `AUDIT_TARGET_TYPES` статичен:** новый домен на беке потребует дописать литерал (append-only). Незнакомый `target_type` в URL до этого будет молча отброшен схемой — записи с новым типом всё равно видны без фильтра.
- **`action` фильтруется точным равенством** — подстрочного поиска на беке нет; regex схемы (`domain.verb`, до трёх сегментов, snake_case) соответствует фактическим call-sites `audit.Logger`.
- **Невалидные `from`/`to` бек молча игнорирует** — фронт-схема отбрасывает их раньше, поведение согласовано.
- **Zod v4** (`package.json`: `^4.3.6`): используемые API (`z.string().uuid()`, `z.enum` по readonly-кортежу, `z.coerce.number()`, `.catch()`, `z.ZodIssueCode.custom`) совместимы и уже применяются в `src/features/glossary/schemas.ts`.
