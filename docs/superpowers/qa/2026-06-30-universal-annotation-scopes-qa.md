# Ручной QA — универсальные anchor-скоупы

Реализация: ветка `worktree-universal-annotation-scopes` (план
`docs/superpowers/plans/2026-06-30-universal-annotation-scopes.md`). Автотесты (vitest)
геометрию Range/выноски/широкий↔узкий/RTL/touch НЕ ловят — это ручная приёмка.

Локально: бэк philosophy-api на :8090 (`make run-local`), фронт `pnpm dev` на :3001,
dev-админ `dev`/`admin12345`. Нужна лекция с инлайн-документом (`?doc=`) и несколькими
комментариями.

## Широкий экран (≥80rem)

- [ ] Выделить фрагмент в ТЕЛЕ КОММЕНТАРИЯ → появляется одна кнопка «Аннотация».
- [ ] Создать аннотацию к комментарию → карточка встаёт в ПРАВУЮ маргиналию у фрагмента,
      выноска-линия тянется к нужному комментарию.
- [ ] Аннотации документа и аннотации комментариев живут в ОДНОЙ правой рельсе, не наезжают
      (глобальный стекинг по Y).
- [ ] UX аннотаций документа НЕ изменился: тумблер подсветки (скрыть/показать), кнопка
      «Добавить аннотацию» (без якоря), тон/акцент карточек, orphan-лейбл «Фрагмент не найден».
- [ ] **Реципрокная навигация документа сохранена (паритет):** клик по подсвеченному фрагменту
      в тексте → активирует+проматывает к карточке; наведение на фрагмент → акцент карточки;
      клик по карточке → проматывает к фрагменту. (Все три направления.)
- [ ] Заякоренные комментарии (ЛЕВАЯ рельса, синий тон) работают как раньше: позиционированные
      превью, выноски, создание комментария из выделения в документе.
- [ ] Выделение ЧЕРЕЗ границу скоупа (из комментария в документ / из документа в комментарий)
      → кнопки аффорданса НЕТ.
- [ ] В скоупе комментария предлагается ТОЛЬКО «аннотировать» (НЕ «комментировать-фрагмент» —
      v1 не пускает якорь комментария в комментарий).
- [ ] Подсветка фрагмента комментария включается/гаснет (общий канал annotation на странице).
- [ ] Сирота: отредактировать тело так, чтобы `exact` не нашёлся → карточка в orphan-зоне рельсы
      с «Фрагмент не найден».
- [ ] Новая аннотация появляется СРАЗУ спозиционированной (не сиротой) — проверка fingerprint
      scopeKey: добавление заметки пересчитывает геометрию без resize.

## Узкий экран (<80rem)

- [ ] Карточки аннотаций комментария текут inline ПОД телом своего комментария (локальность),
      без абсолютного позиционирования и выносок.
- [ ] Аннотации документа на narrow тоже inline (поле схлопнуто), без двойного рендера/мигания.

## RTL / a11y / touch

- [ ] В RTL акцент-бордюр карточки на логической стартовой стороне; выноски зеркалятся.
- [ ] Touch-выделение в комментарии вызывает аффорданс (pointerup/touchend).
- [ ] Tab-порядок карточек в рельсе соответствует визуальному (WCAG 2.4.3 Focus Order).

## Перф

- [ ] Тред с многими комментариями (≥30, часть с аннотациями): нет заметного лага при
      resize/scroll — один проход пересчёта в rail (ResizeObserver на корнях, не N²).
- [ ] Один HTTP на лекцию за аннотациями комментариев (`GET /api/lectures/{id}/annotations?parent_entity_type=comment`,
      React `cache()`), а не N запросов на N комментариев. Проверить в Network-вкладке.
